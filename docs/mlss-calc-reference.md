## 0. 全体演算チェーン（上流 → 下流）

```
[プローブ MCP3424]                          [本体 IM_110.c]
 ADC digit
   │ ①生mV正規化 (ch別)
   ▼
 MCP3424_AD_mV[ch]  ──②移動平均(SADA)──►  UART送信(MS/MD)  ──►  ADC_mV[ch] / ADC_mV_ave[ch]
                                                                     │  Y0=[0]受光, Ref=[1]
                                                                     ▼ ③ A/Dゼロ補正 + 温度補正 + 対数
                                                                   MLSS_ABSS (ABS)
                                                                     ▼ ④ ベース (出荷時2次式×k_No / TR=累乗)
                                                                   FABSS
                                                                     ▼ ⑤ 機差補正 kiza … 【廃止】(§4)
                                                                     ▼ ⑥ スパン校正 (SP_A/B/C, 現場2/3点)
                                                                   MLSS [mg/L] → 表示
```

### ADC チャンネル対応表（真実源: adjust-mode-design.md §204 / protocol-rs232c.md §195-207 / adj_mode_ch）

| ch番号(1始まり) | code index | 信号 | 物理 MCP3424 | 備考 |
|---|---|---|---|---|
| ch1 | `[0]` | MLSS 受光 (Y0) | CH1 | LED PWM 対象 |
| ch2 | `[1]` | MLSS Ref | CH2 | LED PWM 対象 |
| ch3 | `[2]` | SS 受光 | CH3 (SEL3 LOW, PGA×1) | LED PWM 対象 |
| ch4 | `[3]` | SS Ref | CH4 | LED PWM 対象 |
| ch5 | `[4]` | 透視度 | CH3 (SEL3 HIGH, **PGA×2**) | ch3 のゲイン/スケーリング変更値だが **A/DC へ別途入力**。**LED PWM 対象外／ADSpan 調整は対象** |

- **LED PWM 調整の対象 = ch1〜4**（index `[0]`〜`[3]`, MLSS受光/MLSS_Ref/SS受光/SS_Ref）。ch5 は除外。
- **ADSpan 個別 span傾きの対象 = ch1〜5**（index `[0]`〜`[4]`）。**ch5(透視度)も A/DC 別途入力ゆえ ADSpan 要**。
- MLSS で使うのは **ch1=`[0]`(受光 Y0) / ch2=`[1]`(Ref)**。
- 本書では以降「ch番号」はこの 1 始まり、「`[n]`」は code index を指す。

## A. プローブ / 本体 責務分界（設計原則, 2026-07-20 確定）

- **プローブ = AD 値(mV)をそのまま出力**する。補正・計算は **本体** で行う。
- 係数・パラメータのうち **プローブに関するものはプローブ側フラッシュに保存**するが、**補正/計算は本体**。
- **プローブ側の計算に使う調整パラメータは 2 種のみ**:
  1. **`ADC_Span[]`** … 空中出力を 1700mV にする**傾き(span)正規化**基準（第1領域 PARAM）。**2026-07-21 に ADZero オフセットから span 傾きへ変更**。
  2. **LED PWM duty (`LED_Out[0]`)** … 初期値 0.36（第1領域 PARAM）
- **オフセット ADZero（`生mV − ADC_Zero` の減算）は廃止**（2026-07-20〜21 の巻き戻し）。プローブ側計算は span 傾き一本。`ADC_Zero`/`SADZ` は残置・計算未使用。

改訂後のプローブ処理（実質）:
```
プローブ出力 mV = 生mV(digit×LSB) × ADC_SPAN_INI(=1700) / ADC_Span   （LED は duty で駆動するだけ）
本体側        : 出力mV → ABS → 4次式(Mode_CF) → 温度補正 → 機差kiza → スパンSP → 物理量
```
- 空中で `ADC_Span` = その ch の空中生mV を捕捉するので、空中出力 = 生mV × 1700 / 生mV = **1700mV**。原点を通る純粋な傾き調整。
- Mode_CF(4次) 等の**係数はプローブ統合ストア(係数ページ)に保存**するが、**適用計算は本体**（現状どおり）。

> 本原則により、以下 §1 の正規化式は「span 傾き `×ADC_SPAN_INI/ADC_Span`（ADC_SPAN_INI=1700）を採用し、ADZero 減算は撤去」に戻す（2026-07-20 の ADZero 一本化を巻き戻し、ターゲットを 1750→1700 に変更）。

## A-2. 真実源・同期・初期化モデル（2026-07-20 確定、ID-200 の 1Wire EEPROM 流用）

- **真実源 = プローブ flash**（全係数、**現場上書き分も含む**）。本体故障交換時も再調整不要／プローブ交換時は再調整。
- **本体EEPROMミラー**: プローブ領域と**完全に同一配置**＋ **プローブ個体ID** ＋ **最終更新日**（RTC由来）を保持。非権威キャッシュ。
- **本体側は LED PWM / ADC_Span / ADC_Zero(残置) など第1領域パラメータも読み出す**（プローブ個別調整値の確認用に共通領域として保持。正規化計算自体はプローブ側）。
- **プローブ個体ID は既存コマンドで実装済み**（新規不要）: 書=`SID,N`（uint32 10進）／ 読=`RPP` ダンプ中の `Probe ID:` ／ 永続=`WPP`。
  `Probe_ID`(`uint32_t`) は第1領域 PARAM に格納。本体ミラーにこの ID を持たせて突合する。

**起動時 初期化（3層 + ID 判定）**:
```
(1) 変数に既定値をセット
(2) 本体EEPROMミラー（同一配置＋プローブID＋最終更新日）から読み出し上書き
(3) 判定:
    プローブID ≠ ミラーID            → 必ずプローブから読む（別個体。=1Wireのセンサー製造番号に相当）
    プローブID = ミラーID かつ 日付[プローブ ≥ 本体] → プローブから読む（プローブ=真実源、同一もプローブ優先）
    プローブID = ミラーID かつ 日付[本体 > プローブ]  → 本体ミラー維持（前回プローブ書戻し失敗時のみ）→ 再同期
    プローブ読取失敗                 → 本体ミラーで測定継続（耐性）
```
**書き戻し（電源OFF 一括）**: 変更分を **プローブ flash と 本体EEPROMミラーの両方**へ、**同じ ID・最終更新日**で書く。
- 校正/調整は運用中 RAM キャッシュに反映 → OFF時に一括書戻し（flash-busy 窓を測定中にまたがない＝統合ストア不安定の緩和）。
- 急電断で当セッション変更が失われる点は 1Wire と同性質（通常はボタンOFFの制御シャットダウンで書戻し窓を確保）。

## 1. プローブ側: ADC digit → 送信 mV（①生 mV 正規化）

`IM-110_Probe/Core/Src/IM_110.c:205`（ch0-3 通常）:

```
MCP3424_AD_mV[ch] = ( digit × LSB ) × ADC_SPAN_INI / ADC_Span[ch]        （ADZero 減算は撤去, 2026-07-21）
```

- **LSB** = 0.0625 mV/digit（ch0-3, PGA=1）／ 0.03125 mV/digit（ch4, PGA=2, `:199`）
- ch4 のみオフセット項 `+ ADC_CH4_VREF`（=1000mV）が span 前に入る（`:199-200`）: `(digit×0.03125 + ADC_CH4_VREF) × ADC_SPAN_INI / ADC_Span[4]`
- 移動平均（②）: `SADA` 回で平均（既定 30）→ 本体では `ADC_mV_ave[ch]` として届く

### パラメータ（プローブ, `Core/Inc/IM_110.h`）

| 記号 | 役割 | 初期値 | 設定コマンド | 保存 |
|---|---|---|---|---|
| `ADC_Zero[ch]` | **【残置・計算未使用】** AD ゼロ点（旧オフセット。span 復活で不使用） | `ADC_ZERO_INI=0.0` | **SADZ,ch** (`:689`) | WPP→flash |
| `ADC_Span[ch]` | **AD スパン（空中の生 mV = 傾き基準）**。MLSS AD0 の能動側 | `ADC_SPAN_INI=1700.0` | **SADS,ch** (`:720`) | WPP→flash |
| `ADC_SPAN_INI` | 正規化ターゲット（空中出力の目標 mV） | **1700.0**（旧 1750, 2026-07-21 変更） | 定数 | — |
| `ADC_SPAN_S_INI` | 出荷時スパン（live 未使用） | 1850.0 | 定数 | — |
| `LED_Out[0]` | **LED PWM duty**（別機能, ALDA/手順2） | `LED_OUT_INI=0.36` | **SP,duty** (`:509`) | WPP→flash |
| `ADC_CH4_VREF` | ch4 IN- 1V オフセット補正 | 1000.0 mV | 定数 | — |

- **SADS,ch**（`:726`）【能動・MLSS AD0】: `ADC_Span[ch] = 生mV`（空中の生値, +VREF ch4）… **取得ゲート 生mV 1400〜2048mV, 範囲外 NG**。空中でこの ch を span 基準に採用 → 空中出力 = 1700mV。
- **SADZ,ch**（`:689`）【残置・計算未使用】: `ADC_Zero[ch]` を設定するが正規化式では不使用。

> 意味: プローブは「生 mV を空中基準 `ADC_Span` で割って ADC_SPAN_INI 倍」＝ **AD 値を空中 = ADC_SPAN_INI(=1700) となる傾きに正規化**して送る（原点を通る傾き調整）。
> 空中で受光が 1700mV 付近になるよう LED duty を合わせる（手順2/ALDA）のは、この正規化が素直に効く動作点にするため（LED 目標も 1700 に整合）。

## 2. 本体側: 生 mV(Y0, Ref) → 吸光度 ABS（③）

`IM-110/Core/Src/IM_110.c:995` `MLSS_FABSS_from_raw_mv()`:

```
adc_mv = Y0 − MLSS_ADZR                         (≤0 は 0.001 にクランプ)         :997,1000
d      = Ref − MLSS_Coef_ADZR[1]   (= Ref − refZR)                              :1003
I(0)   = MLSS_ZR + d × ( TempC[0] + d × TempC[1] )   (I0≤0 は MLSS_ZR)           :1004,1005
ABS    = log10( I(0) / adc_mv )                                                  :1006
```

仕様書 付録A（`IM-110プログラム仕様書.md:3518`）と一致:
```
ABS = Log( (MLSSZR + (現在のref出力 − refZR) × ref補正傾きB) / (Y0 − ADZR) )
```

### 用語対応（仕様書 ↔ コード）

| 仕様書 | コード（本体） | 意味 | 初期値 / 由来 |
|---|---|---|---|
| Y0 | `ADC_mV_ave[0]`（瞬時=`ADC_mV[0]`） | MLSS 受光 生 mV | プローブ送信 |
| 現在のref出力 | `ADC_mV_ave[1]` | MLSS_Ref 生 mV | プローブ送信 |
| **MLSSZR** | `MLSS_ZR` | ゼロ基準（清水/空中の分子基準 mV） | `INI_MLSS_ZR = 1850.0` |
| **ADZR** | `MLSS_ADZR` (= `MLSS_Coef_ADZR[0]`) | 受光暗時ゼロ | `INI_MLSS_ADZR = 0.0` |
| **refZR** | `MLSS_Coef_ADZR[1]` | Ref 基準（20℃清水 Ref） | 統合ストア（温度校正 ATCF が設定） |
| **ref補正傾きB** | `MLSS_Coef_TempC[0]`（B2=`[1]`） | Ref 温度補正傾き | 統合ストア（未校正=0 → I(0)=MLSSZR） |

- **温度補正が未校正**（TempC=0）なら `d×(…)=0` ⇒ `I(0)=MLSS_ZR`（= MLSSZR そのもの）。
- 上記は**現行コード**。↓ の改訂で暗時 ADZR を廃止する。

### 改訂（2026-07-20 確定）: 暗時 ADZR 廃止・オフセット2層

**暗時 ADZR（`Y0 − ADZR` の減算）を廃止**。測定式は:
```
I(0) = MLSSZR + (Ref − refZR)·(B + (Ref−refZR)·B2)     ← Ref温度補正（20℃換算, §8-T）
ABS  = log10( I(0) / Y0 )                               ← 分母は Y0 そのまま（暗時減算なし）
```
受光に効くオフセットは**2層・併用**:

| # | 名称 | 場所 | 役割 | 書換 |
|---|---|---|---|---|
| ① | **プローブ空中AD調整 (ADC_Span[], span傾き)** | 第1領域 PARAM, per-ch | Y0 を **空中→1700** に傾き正規化（AD動作点・飽和回避）。2026-07-21 に ADZero オフセット→span 傾きへ | ADBOAD/UART のみ |
| ② | **清水中オフセット = MLSSZR** | 本体 | 測定ゼロ基準（清水 Y0 = 0mg/L 点）。`ABS=log10(MLSSZR/Y0)` の分子 | 下記2系統 |

- **② MLSSZR の2系統**（＝従来抜けていた「ゼロ校正係数」を補完）:
  - **出荷時清水中オフセット**（工場, ADBOAD/UART）＝ MLSSZR 初期値。
  - **通常校正モードのゼロ校正**（現場, ユーザ）＝ 清水 Y0 を取得し **出荷時値に置き換わる値**として MLSSZR を更新。
  - No.21-30 と同構図（出荷時=初期値／校正=上書き／初期化で出荷時へ復帰）。
- **MLSSZR は固定 1700 ではなく「清水中 Y0」**（空中1700 より少し低い）。← 従来 §7 の 1850/1750 齟齬の正体。
- 清水中: `MLSSZR=Y0_清水` ⇒ `ABS=log10(Y0_清水/Y0_清水)=0` ⇒ MLSS=0。濁ると Y0↓ ⇒ ABS>0（単調増加）。
- ①(空中1700 span正規化) と ②(清水ゼロ基準) は**目的が別**ゆえ併用。SS/TR も同様（TR は ch5、温度補正は SS 流用）。

## 3. 1次演算: ABS → FABSS（④相関式 Mode_CF）

`IM_110.c:1007-1013`（6次までの多項式, ホーナーではなく明示展開）:

```
FABSS = Σ(k=0..6) MLSS_Mode_CF[MLSS_MODE][k] × ABS^k
```

- `MLSS_MODE` = 選択中の相関式テーブル index（0=No.1 … 19=No.20 … 29=No.30）
- **現状テーブルは全て「1次直線」**（`MLSS_Mode_CF[..][1]=傾き`, 他 0）⇒ 実質 `FABSS = C1 × ABS`
- **No.1-20 = FW 固定・校正不可**（等間隔グラデ, C1: No.1=21315 … No.11=42494.4572(=従来黒No.5) … No.20=61555.97）
- **No.21-30 = 校正可**（初期 C1=21315=No.1 傾き, ADBOAD 校正で統合ストアの modecf を上書き → `Apply_Coef_To_Live` で反映, `:956`）

（テーブル全体は `IM_110.c:109-143`）

## 4. 機差補正 kiza（⑤工場, `IM_110.c:1015` / `sel_kiza :932`）

```
FABSS ×= sel_kiza(mode=0, MLSS_MODE)
sel_kiza = MLSS_Coef_Kiza[vi]   (vi = MLSS_MODE − 20, 0.0 のときは 1.0 中立)
```

- **1 点比例**（`倍率 = 基準器 mg/L ÷ 1次演算後 mg/L`）で基準器に合わせる工場調整（design G）。
- per-eq（相関式ごと）。統合ストア保存。旧 `FABS_SPAN`（`MLSS_FABS_SPAN`）は**廃止**（式には未使用、初期 1.0 のまま残存）。

### 決定（2026-07-20）: **kiza 廃止**
- 器差（個体差）は次の 3 調整で吸収し、量産個体の数値誤差を統一する:
  1. **LED PWM**: ch1〜4 の最大値 = 1700mV
  2. **ADSpan (span傾き)**: 空気中 = 1700mV
  3. **出荷時3点調整**: ゼロ = 清水中出力（= MLSSZR ②, §2改訂）／ 2,3点 = **基準器(IM-110)の指し示す値**
- 出荷時3点調整が基準器合わせを兼ねるため、別途 1 点比例の `kiza` は**不要＝廃止**。
- measurement チェーンは `ベース → span校正` の 2 段（`×kiza` を削除）。

## 5. スパン校正 SP（⑥現場, `IM_110.c:1027`）

```
MLSS = MLSS_SP_A × FABSS² + MLSS_SP_B × FABSS + MLSS_SP_C
```

- 現場の 2点/3点後校正（ADCAL/MCAL）で `SP_A/B/C` を作る。機差補正の上に乗る多段（design G）。
- 初期値（`IM_110.c:93-95`）: `SP_A=0.0`, `SP_B=1.0`, `SP_C=0.0` ⇒ 未校正時 `MLSS = FABSS`。
- スパン校正係数（仕様書 `:3482`）: `SP_B = スパン校正値(mg/L) ÷ 1次演算後(mg/L)`
- 瞬時値 `MLSS_inst` も同式で `ADC_mV[0/1]`（瞬時）から算出（界面バー用, `:1029-1032`）。

## 7. 現状の不整合（要判断・修正対象の候補）

1. **【解決 2026-07-21】MLSSZR デフォルト 1850 vs 正規化ターゲット 1700 の不一致**
   - プローブ: `ADC_SPAN_INI = 1700`（送信 mV は空中 1700 基準に span 正規化, 2026-07-19 に 1850→1750, 2026-07-21 に 1750→1700）
   - 本体: `INI_MLSS_ZR / INI_SS_ZR / INI_TR_ZR = **1700**`（旧1850）へ更新済み（[IM_110.h:11/19/27](../../IM-110/Core/Inc/IM_110.h#L11)）。デフォルト==空中ターゲットの対応を復元。
   - ⇒ 電源投入デフォルトが 1700 スケールで一致。**実運用の MLSSZR は AZC(清水ゼロ校正)が空中正規化後の Y0 で上書き**（デフォルトは校正前プレースホルダ）。
   - ⚠ **残: 実機の出荷時校正係数（旧1850/1750前提で取得したもの）は 1700 スケールで取り直しが必要**（コードでなくデータ側。実機での再校正で解消）。

2. **「空中ターゲット mV」は 2 系統に登場する（用語の衝突源）**
   - **LED duty（PWM/SP/ALDA, 手順2）**: 空中受光を 1700±? に合わせる能動サーボ（別機能）。
   - **MLSS ゼロ調整（MLSSZR）**: 生 AD の分子基準。清水/空中の受光 mV を基準として取得。

## 8. 基板調整 AD調整 — 改訂（2026-07-20）: プローブフラッシュ2領域で分担

プローブフラッシュ（`docs/probe-flash-map.md`）の 2 領域で役割を分ける。**SS / TR も共通の考え方**。

### プローブ／出荷時 調整の順序（2026-07-20 確定）
1. **LED PWM 調整**（明るさ・ch1〜4 max→1700）
2. **ADSpan[] 個別 span傾き**（ch1〜5 → 空中1700）
3. **Ref 温度補正**（5/20/35℃ 清水 Ref → 2次式, 20℃換算）← §8-T
4. **出荷時3点調整**（ABS→mg/L の **2次式** = No.1 基準）← 温度補正確定後に実施

理由（3→4 の順）: 出荷時3点調整の ABS は温度補正込みの `I(0)`（§2）から出るため、
**先に温度補正を確定**して各点 ABS を 20℃換算した状態で焼き込む。逆順は非20℃で焼いてしまいズレる。

### 第0ステップ: LED PWM 調整（＝1700mV 収束機能, ALDA 相当）
- **目的**: 空中で **ch1〜4 の AD が 2048mV（＝16bit AD 満スケール, 振り切り）に達しない**よう LED 出力を下げる。
- **基準**: **ch1〜4 の生 mV で最も高いものが 1700mV** になるよう LED duty を調整（比例＋収束反復のサーボ）。span 正規化ターゲットと一致（2026-07-21 に 1750→1700）。
- **保存先**: `LED_Out[0]`（第1領域 PARAM）。
- これが「1700mV に収束する機能」（先に AD0 と混同していたもの。**別機能**）。
- 実装参照: 本体 [Adj_LED_Auto](../../IM-110/Core/Src/IM_110.c#L1553)（`ALDA`）。
- **対象 ch = ch1〜4（code index `[0]`〜`[3]`）**＝ MLSS受光/MLSS_Ref/SS受光/SS_Ref。**ch5(透視度 `[4]`)は除外**（§0 対応表）。


| 領域 | ページ | 保持するもの | 本 AD 調整での役割 |
|---|---|---|---|
| **第1領域 = PARAM** | Page62 (`PARAM_AD`) | `ADC_Span[]`, `LED_Out[]` 等 | **ADSpan = 空気中出力を 1700mV にする傾き基準**（①・2026-07-21 に ADZero オフセットから変更） |
| **第2領域 = 統合ストア(STORE)** | Page63 (`STORE_AD`) | 校正係数(統合ストア) | **出荷時2次式(No.1基準)**＋**出荷時清水中オフセット(MLSSZR初期値, ②)**＋Ref温度補正＋No.21-30 |

> 受光ゼロは 2 層併用（① 空中1700=ADSpan傾き / ② 清水ゼロ=MLSSZR）。詳細は §2 改訂。暗時 ADZR は廃止。

### 第1領域: ADSpan（＝画面「MLSS AD0」）
- **ADSpan** = **空気中**で受光出力を **1700mV にする傾き(span)基準**。プローブフラッシュ**第1領域(PARAM)**。空中の生 mV を `ADC_Span[ch]` に記録し、`出力=生mV×1700/ADC_Span` で正規化 → 空中 1700mV。**2026-07-21 に ADZero オフセット→span 傾きへ変更**。
- **対象 ch = ch1〜5（index `[0]`〜`[4]`）を個別に**。ch5(透視度)も A/DC 別途入力ゆえ ADSpan 要（LED PWM は ch1〜4 のみ）。
- ID-200T ADZERO（[Calc.c:432](../../IM-110/Core/Src/Calc.c#L432) `calc20` case0 `ADzr[0]=R`）と類似操作だが、ID-200T は **0mV オフセット**、IM-110 は **空中1700mV への傾き正規化**。
- 取得ゲート: **生 mV 1400〜2048mV**（範囲外は取得しない＝NG）。
- 画面タイトル **「MLSS AD0」**（基板調整側。校正側「MLSS 0」とは別物）。

### 廃止
- **`ADZero`（オフセット `生mV − ADC_Zero`）・`ADMiddle[]` は無し（廃止・未使用）**。
  → 2026-07-20 に一度 ADZero 一本化したが、2026-07-21 に **span 傾き正規化（`× ADC_SPAN_INI / ADC_Span`, `IM_110.c:205`）へ巻き戻し**。`ADC_Zero`/`SADZ` はコマンドのみ残置・計算未使用。
  曲線/高次項は第2領域の 4次式（出荷時3点校正）が担う。

### 第2領域: 出荷時 3 点校正係数（4次式）— プローブ調整の第3ステップ
プローブ調整（LED PWM → ADSpan）の後に実施する。

- **動作は現行「MLSS 校正モード, 相関式 No.21〜30」と全く同じ** 3 点校正機構
  （[adj_fit_poly](../../IM-110/Core/Src/IM_110.c#L1244) 多項式フィット → `*_Coef_ModeCF` → `WCM` → `WCFC` 確定）。
- **この校正機能を ADBOAD にも追加**する（現状は CAL/校正モードのみ）。**MLSS / SS / TR すべて対象**。
- **出荷時補正係数 = MLSS / SS / TR それぞれ 1 式ずつ持つ**（＝出荷初期値係数）。
  - **実質 2 次式**（3 点で一意に決まるのは 2 次まで）。格納は `modecf[7]` 枠（最大6次）だが **3 次以上は 0**。
    ＝「係数枠が 4 次まであるだけ」で実体は 2 次。
  - 経緯: 元の MLSS白 6点→4次式（付録A）は実験式。**最終的に 1 次傾き補正（相関式 No.13 ≒ ゲイン2.19 相当）に集約**したため、
    出荷時は **3点→2次式で確定**（4次は使わない）。

### 相関式 No. モデル（改訂・決定: (B) 併存）

- **相関式係数は今回から「傾きのみ（1次スケール）」に簡素化**（元は独立した 3 点調整の 4 次式係数だった）。
- **出荷時 MLSS 3 点調整係数 ＝ 相関式 No.1 相当の 4 次係数**（＝基準カーブ）。
- **No.1〜20**: この **No.1（基準 4 次式）を基準に、傾きを順次大きくしていく**（No.1=基準, No.2→No.20 で傾き増）。
  - 各相関式が持つのは **傾き（スケール）係数のみ**。基準の 4 次カーブ形状は共通で、傾きだけが No. で変わる。
- **No.21〜30**: **独立した 2 次式**（ユーザ 3 点校正で作成、基準の代わりに使う）。**傾き(ゲイン) k は 1.00 固定**
  （傾きは 2 次式係数に内包されるため追加ゲイン不要）。← (B) 併存。

**統一形（No.1〜30 共通）**:

## 8-S. SS / TR への適用

- **考え方は MLSS と共通**: ADSpan 個別（SS=ch3/ch4, TR=ch5）／LED PWM（SS=ch3/ch4、TR=ch5 は対象外）／
  出荷時基準式／Ref温度補正（**TR は SS を流用**）／No.21 初期値=出荷時係数コピー。
- **SS / TR は相関式 No. が無い＝No.21 の単一式のみ**（No.1〜20 のゲイン選択なし）。
  単一の校正可能式ゆえ、**校正モードで ゼロ校正＋2点/3点校正が自然に実施可能**。

### TR(透視度) のカーブ形 — 累乗 → 2次補正（仕様書記載もれ、2026-07-20 定義）
- **出荷時ベース = 累乗式** `cm₀ = a·ABS^b`（[TR_FABSS_from_raw_mv](../../IM-110/Core/Src/IM_110.c#L1092)、AMCP=累乗フィット、a,b=TR modecf）。
- **その後に 2 次補正式**: `cm = A·cm₀² + B·cm₀ + C`（校正で作る。MLSS/SS の SP span校正と同一の2次）。
- **校正方法は MLSS と同じ**（2/3点、基準器表示値を入力してフィット）:
  例) 90cm 相当サンプル→基準器 92cm 入力=スパン点、45cm 相当→基準器 45cm 入力=中間点 → (累乗出力, 基準器cm) を2次フィット。
- 現行コードは既に `TR_FABSS(累乗) → TR_SP_A·f²+TR_SP_B·f+TR_SP_C`（[adj_read_live](../../IM-110/Core/Src/IM_110.c#L1458)）＝**実装済み構造**。

| | 出荷時ベース | 上に乗る2次補正(校正) |
|---|---|---|
| MLSS/SS | 2次式(No.1基準) × k_No | SP span校正 `A·f²+B·f+C` |
| TR | 累乗 `a·ABS^b` | 同じ2次 `A·cm₀²+B·cm₀+C` |

- **全モード共通構造**: 「ベース(2次/累乗) → 2次補正(校正)」。ベース形が違うだけで**上に乗る2次補正・校正操作は完全共通**。

## 8-T. Ref 温度補正（出荷時3点調整の前に実施）

- **機能**: **5 / 20 / 35℃ の清水中 Ref 出力**で多項式を作り、それで **MLSS・SS を常に 20℃換算出力に補正**する。
  Ref を温度プロキシに使い、運転時に温度計不要（Ref は測定 ch の1つ）。
- **算出方法**: 出荷時3点調整と**同じフィット関数** [adj_fit_poly](../../IM-110/Core/Src/IM_110.c#L1244) を流用。
  **3 点ゆえ 2 次式**（`B, B2` の2係数＋refZR）。
- **【重要】相対 mV で式を立てる**（原本仕様書に準拠）: 測定式は `I(0)=MLSSZR + B·d + B2·d²`（`d = Ref − refZR`）＝
  **相対 Ref `d`（20℃基準からの変化幅）の多項式**。よって 2 次フィットは **x=`d=Ref−Ref20`, y=`Δ受光=受光−受光20`** の
  **相対 mV・原点通過**で解く（20℃点=(0,0)、5℃/35℃の2点で B,B2 を決定）。**絶対 mV で 2 次を立ててはいけない**
  （係数が平行移動不変でないため測定式と不整合）。
  - 現行 [adj_calc_tempc](../../IM-110/Core/Src/IM_110.c#L1364) は **1 次(傾き B)** で、傾きは平行移動不変ゆえ絶対/相対同値・実質相対ベース（refZR=ref20 を別持ち）。
    2 次化の際に上記「相対 mV・原点通過」を厳守する。
- **式への反映**（§2）: `I(0) = MLSSZR + (現ref − refZR)·(B + (現ref − refZR)·B2)`。
  `refZR` = 20℃ 清水 Ref、`B=tempc[0]`、`B2=tempc[1]`。未校正(0)なら `I(0)=MLSSZR`（補正中立）。
- **TR(透視度)** は **SS の温度補正を流用**（同一光学 ch・Ref 共通、既存設計）。TR 固有の温度係数は持たない。
- **順序**: LED PWM → ADSpan → **本補正** → 出荷時3点調整（§8 順序表）。

### 命名の区別（混同禁止）
| 名称 | 側 | 実体 / 保存先 |
|---|---|---|
| **MLSS AD0** | 基板調整 | 受光 AD の 空中1700 span傾き → 第1領域 `ADC_Span[]` |
| **MLSS 0** | 校正 | 出荷時3点校正係数(4次式) → 第2領域 統合ストア `modecf[7]` |

## 10. 水深補正（2026-07-20 定義）

- **原点**: **電源ON 直後の大気圧**を原点（相対値）。自動ゼロ。現行 [Calc_Depth](../../IM-110/Core/Src/IM_110.c#L1152) で
  初回 `Depth_offset = P` として実装済み。
- **換算**: `Depth[m] = (P − P_atm0) × k_depth`。`P` = プローブ **HSPPAD143A**（PTDET, hPa）→ 本体 `ADC_mV[5]`。
- **校正**: **1 点スロープ校正**。ADBOAD スイッチ / UART デバッグコマンドで、**6m 相当の加圧時に 6.0m 表示**になるよう
  `k_depth`（傾き）を調整。オフセットは自動ゼロなので**傾きのみ**。
- **初期値**: 現行の固定 100hPa/m 相当（`k_depth = 1/100`）。現行の `Depth=(P−offset)/100.0` を `×k_depth` へ置換。
- **保存/権限**: `k_depth` はプローブ側保存、**ADBOAD/UART のみ書換**（他のプローブ調整係数と同一扱い）。

## 11. 残・実装事項（仕様は確定、作業は別途）

- **フラッシュ配置**: 現行 第1領域(PARAM)＋第2領域(係数ページ) を **1 領域に統合**する方針（次トピックで詰める）。
  併せて全プローブ調整係数の配置を確定。※ 係数格納フラッシュの通信不安定（`probe-flash-map.md` §不安定）の恒久化とも関係。
  - **32B ページ共通レコード方式（2026-07-20 決定）**: 本体EEPROM の 1 ページ=32B（7float+4B）構造を
    そのままプローブflashのデータ構造に流用し、**本体EEPROM ⇄ プローブflash をバイト単位で完全一致**させる。
    → §A-2 の同一配置ミラーを `memcpy`/ブロック転送で実現、struct 定義・シリアライズ層を共通化。
    - 成立根拠: 両 MCU とも little-endian ARM + IEEE754 単精度 → float 表現がバイト一致。
    - 共通化範囲: データ配置＋シリアライズのみ（低レベルアクセスは本体=M95256 SPI EEPROM / プローブ=内蔵Flash で別）。
    - 実装注意: 構造体は明示レイアウト（4B境界/packed）で padding 差を排除。プローブは2KBページ消去→全レコード書込（OFF時一括書戻しと整合）。
- **格納が必要な係数（統合対象の棚卸し）**:
  - ADC_Span[ch]（空中1700 span傾き, 5ch）／ LED PWM duty ／ k_depth
  - 出荷時清水オフセット = MLSSZR 初期値（mode別）／ Ref温度補正（refZR, B, B2; MLSS/SS）
  - 出荷時2次式（No.1基準, MLSS/SS）＋ TR 累乗(a,b) ／ k_No 枠(No.1-20) ／ No.21-30 独立2次式(各10)
  - 現場校正: MLSSZR上書き値 ／ SP_A/B/C（MLSS/SS/TR）
- 統合ストア(§12)へ吸収し、protocol §3.2.6 の 32B ページ転送(RPG/WPG/WSC)で読み書き（旧 ver2→ver3 拡張は廃止）。

## 12. フラッシュ レコード割付（2026-07-20 確定案）

**方針**: 1 機能 = 32B ページ単位で分離。**32B 共通レコード**で プローブflash ⇄ 本体EEPROMミラー を **相対レコードN で完全一致**
（`本体EEPROM[ミラー基準+N] == プローブflash[領域先頭+N]`）。絶対ページNo.は別（本体EEPROMは header/共通設定/界面/測定履歴 等
本体固有データも持つため、ミラーは連続サブブロックに置く）。既存の本体EEPROM疎レイアウト(page19-60,2page/相関式)は本配置に**置換**。

### 12.1 新品（工場出荷前）既定値 — **単一ソース = `probe_store.h store_set_new_probe_defaults()`**

新品化（プローブ `RPF` / 本体ミラー未同期時の seed）で書き込む既定イメージ。**係数枠を 0 埋めしてはならない**:
0 埋めだと本体 `store_unpack_to_globals()` が基準カーブと ZR まで 0 上書きし、`I(0)=0 → log10(0)=−∞`、
続く `eval_modecf(全0係数, −∞)` が `0×∞=NaN` となって MLSS/SS が **NaN**（測定破綻）になる。
新品でも「空中 ≈0・清水近傍で微小値」の健全出力にするため、下表の初期係数を必ず入れる。

| ページ | フィールド | 新品既定 | 由来（本体 live INI と同値） |
|---|---|---|---|
| 0 ヘッダ | magic/ver / probe_id / last_update / product_name | `IM2S`/1 / `0xFFFFFFFF` / 0 / `"IM-110 "` | 新品＝ID未割当・日付未設定 |
| 1 共通 | adc_span[5] / led_out / k_depth | 1700.0 / 0.36 / 1.0 | `ADC_SPAN_INI`/`LED_OUT_INI` |
| 2 設定 | SADA/SADC/選択No./Meas_Mode | 0（現状 unpack/apply 非配線・中立） | — |
| 3/10 MLSS/SS ゼロ温度 | zr_ship / zr_field | **1700.0 / 1700.0** | `INI_MLSS_ZR`/`INI_SS_ZR`（`i0=0` 回避の要） |
| | refzr / b / b2 | 0 / 0 / 0 | 温度補正 中立 |
| 4/11 MLSS/SS ベース校正 | c0,c1,c2 / sp_a,sp_b,sp_c | **0,21315.0,0 / 0,1.0,0** | `*_Mode_CF[0]`(No.1傾き) / `*_SP_*`(恒等) |

- **バイト一致契約**: プローブ flash（`store_set_defaults` → `store_set_new_probe_defaults`）と本体 EEPROM ミラー seed（`mirror_ensure_valid` → 同関数）は**同一ビルダー**で同一 512B を組む（§A-2）。値の変更は `probe_store.h` の `STORE_DEF_*` 一箇所のみ。

# IM-110 プログラム仕様書

> IM-110T 本体・プローブ両基板のプログラム動作仕様。**セクションA=本体、B=プローブ、末尾に付録。**

> **作成方法**: 本体(A)は 77サーバーの Devstral(24B) が本体FWの描画コード `Display.c` と呼出側 状態遷移 `Normal.c`/`Setting.c`/`Adjust.c` を読み、実装から画面仕様・遷移を執筆。付録は `Calc.c`／補正式手順／`Eeprom.h` から生成。プローブ(B)は `protocol-rs232c.md` から生成。Claude Code が真実源と突き合わせ検証。文体はである調に統一。

> 画面図はモノクロLCD(400×240)。全図は `docs/specs/assets/im-110/`。

# セクションA：IM-110T 本体


## 5.1 起動画面

<sub>実装: 描画 `disp_STRTDSP`, `disp_STRTDSP2` / 遷移 呼出側 case</sub>

起動画面は、電源スイッチによる起動直後に表示される画面である。描画は `disp_STRTDSP` が行い、製品名・ファームウェアバージョン・製造者名を表示する。続いて `disp_STRTDSP2` により、校正画面および設定メニュー画面へ入るための操作説明画面を表示する。

`disp_STRTDSP` の表示要素は次のとおりである。

- 製品名「IM-110」（`msg_IM110`）を画面上部に表示する。
- ファームウェアバージョンを表示する。値は `FW_VER1` を 100 で除した数値で、小数第 2 位まで表示する。左側に「Ver.」（`msg_Ver`）の見出しを添える。
- 製造者名「飯島電子工業株式会社」（`msg_iijima`）を下部に表示する。

`disp_STRTDSP` は、起動の経路に応じて次の 3 通りの文脈で呼び出される。いずれの場合も起動画面を表示したうえで、LCD 自動消灯用のタイマーを設定する。

- 通常起動（`STRT_1`）では、測定レンジを設定したのち起動画面を表示する。あわせて保存済みのロガーデータを全件読み込み、読込に失敗した場合は EEPROM エラーフラグを立てる。表示後は操作説明画面へ進む状態（`STRT_4`）へ遷移する。
- 校正メニューからの起動（`CALDSP_1`）では、起動画面を表示したのち記録（MEM）スイッチまたは履歴（DISP）スイッチの押下状態を確認し、`CALDSP_2` へ遷移する。
- 設定メニューからの起動（`SETDSP_1`）では、起動画面を表示したのち `SETDSP_2` へ遷移する。

`disp_STRTDSP2`（操作説明画面）は状態 `STRT_5` で表示され、次の 2 行の操作説明を提示する。表示後は LCD 自動消灯用タイマーを設定し、`STRT_2` へ遷移する。

- 上段には「校正」「画面へ」（`msg_cal_menu` + `msg_toscr`）と記し、記録（MEM）スイッチ（`icon_MEM`）と電源スイッチ（`icon_POWER`）を押しながら起動する旨（`msg_push1` / `msg_push2`）を、各スイッチアイコンとともに表示する。
- 下段には「設定メニュー」「画面へ」（`msg_setting_menu` + `msg_toscr`）と記し、履歴（DISP）スイッチ（`icon_DISP`）と電源スイッチ（`icon_POWER`）を押しながら起動する旨を、各スイッチアイコンとともに表示する。

なお、いずれの描画関数も LCD 表示バッファの初期化（`lcd_init`）に失敗した場合は、描画を行わず異常終了する。

## 5.2 測定メニュー

<sub>実装: 描画 `disp_M_MENU` / 遷移 呼出側 case</sub>

測定メニューは、測定に関する各種操作の入口となる選択画面である。描画は `disp_M_MENU` が行い、4 つのメニュー項目を格子状に配置して表示する。

表示要素は次のとおりである。

- 画面上部にタイトル（`t_meas_menu`）を表示する。
- 右上に電池残量アイコン（`LS027_disp_battery_icon`）を表示する。
- 日付・時刻を表示する（`LS027_disp_datetime`）。
- 画面左側にスイッチ操作の凡例を表示する。履歴（DISP）スイッチのアイコンに対して「長押しで前の画面へ」（`icon_l_return`）と「選択」（`icon_select`）、記録（MEM）スイッチのアイコンに対して「決定」（`icon_decision`）を並べて示す。
- メニュー項目を 2 行 2 列で表示する。左上が「測定モード」、右上が界面測定モード（`Meas_Mode == MEAS_MODE_DEPTH`）のときは「界面設定」、それ以外のときは「相関式」、左下が「校正」、右下が「設定」である。

各メニュー項目は、選択中を示す引数 `select` と点滅制御の引数 `fl` の組み合わせに応じて、強調表示（`b_` 系アイコン）と通常表示（`w_` 系アイコン）を切り替える。選択中の項目は点滅を伴って強調表示される。項目の並び順は、測定モード（`select==0`）、界面設定または相関式（`select==1`）、校正（`select==2`）、設定（`select==3`）である。

スイッチ操作については、凡例アイコンが示すとおり、履歴（DISP）スイッチを押すと選択項目が移動し、記録（MEM）スイッチを押すと選択中の項目を決定し、履歴（DISP）スイッチを長押しすると前の画面へ戻る。

呼出側では、状態 `M_MENU_2` で本画面を表示する。電池残量を更新（`flash_calc(FLASH_BAT, ...)`）したうえで `disp_M_MENU` を呼び出し、表示に成功すると点滅フラグを反転させて表示更新用タイマーを設定し、`M_MENU_3` へ遷移する。選択項目の移動・決定・前画面復帰を処理する状態（`M_MENU_3` 以降）は本スライスに含まれないため、各スイッチ押下による具体的な遷移先は要確認である。

なお、LCD 表示バッファの初期化（`lcd_init`）に失敗した場合は、描画を行わず異常終了する。

## 5.3 モード切替

<sub>実装: 描画 `disp_M_CHANGE` / 遷移 呼出側 case</sub>

モード切替は、測定項目（MLSS／SS／透視度）を選択・切り替えるための画面である。描画は `disp_M_CHANGE` が行い、引数 `flg` の値によって「選択画面」と「切替確定後のメッセージ画面」の 2 通りの表示を切り替える。

いずれの表示にも共通する要素は次のとおりである。

- 画面上部にタイトル（`t_measure`）を表示する。
- 右上に電池残量アイコンを表示する。
- 日付・時刻を表示する。
- 画面左側に、履歴（DISP）スイッチのアイコンと「選択」（`icon_select`）、記録（MEM）スイッチのアイコンと「決定」（`icon_decision`）の凡例を表示する。

`flg == 0` のときは選択画面を表示し、「MLSS」「SS」「透視度」の 3 つのアイコンを配置する。各アイコンは選択中を示す `select` と点滅制御の `fl` の組み合わせに応じて、強調表示（`b_` 系）と通常表示（`w_` 系）を切り替え、選択中の項目が点滅して強調される。項目の並びは MLSS（`select==0`）、SS（`select==1`）、透視度（`select==2`）である。なお「界面」の選択アイコンは廃止されており、界面のしきい値は設定メニュー側で設定する仕様に移行している。

`flg` が 1 以上のときは、切替確定後のメッセージを画面中央に表示する。`flg` の値に応じて MLSS（`msg_MLSS`）、SS（`msg_SS`）、透視度（`msg_Transparency`）、界面（`msg_Interface`）のいずれかのメッセージを表示する。

スイッチ操作については、凡例アイコンが示すとおり、履歴（DISP）スイッチを押すと選択項目が移動し、記録（MEM）スイッチを押すと選択中の測定項目を決定する。

呼出側の状態遷移は次のとおりである。

- `M_CHANGE_1` では、電池残量を更新したうえで選択画面（`flg == 0`）を表示する。表示に成功すると点滅フラグを反転させ、表示更新用タイマーを設定して `M_CHANGE_2` へ遷移する。
- `M_CHANGE_3` では、切替確定後のメッセージ画面を表示する。このとき `flg` には選択中モード（`Meas_Mode`）に 1 を加えた値を渡すため、選択したモードに対応するメッセージが表示される。表示に成功すると、LCD 自動消灯用タイマーと点滅用タイマーを設定したうえで、測定モードの切替処理（`Change_Meas_Mode`）を実行し、切替後のモードに対応する履歴バンクを読み込み（`eep_read_history`）、安定判断をリセット（`reset_stable`）して `M_CHANGE_4` へ遷移する。

選択画面上で記録（MEM）スイッチにより決定してから `M_CHANGE_3` へ至る間の処理を行う状態（`M_CHANGE_2`）は本スライスに含まれないため、決定操作から確定メッセージ表示へ至る具体的な遷移は要確認である。

なお、LCD 表示バッファの初期化（`lcd_init`）に失敗した場合は、描画を行わず異常終了する。



## 5.4 測定モード設定

<sub>実装: 描画 `disp_MEASMODE` / 遷移 呼出側 case</sub>

測定モード設定は、通常測定とデータロガー測定のいずれかを選択・設定するための画面である。設定メニュー配下から呼び出される。描画は `disp_MEASMODE` が行い、引数 `flg` の値によって「選択画面」と「設定確定後のメッセージ画面」の 2 通りの表示を切り替える。

いずれの表示にも共通する要素は次のとおりである。

- 画面上部にタイトル（`t_measmode`）を表示する。
- 右上に電池残量アイコンを表示する。
- 日付・時刻を表示する。
- 画面左側に、履歴（DISP）スイッチのアイコンと「選択」（`icon_select`）、記録（MEM）スイッチのアイコンと「決定」（`icon_decision`）の凡例を表示する。

`flg == 0` のときは選択画面を表示し、「通常」（`select==0`）と「ロガー」（`select==1`）の 2 つのアイコンを上下に配置する。各アイコンは選択中を示す `select` と点滅制御の `fl` の組み合わせに応じて、強調表示（`b_` 系）と通常表示（`w_` 系）を切り替え、選択中の項目が点滅して強調される。

`flg` が 1 以上のときは、設定確定後のメッセージを画面中央に表示する。`flg == 1` のときは通常測定設定のメッセージ（`msg_normal_set`）、`flg == 2` のときはロガー測定設定のメッセージ（`msg_logger_set`）を表示する。

スイッチ操作については、凡例アイコンが示すとおり、履歴（DISP）スイッチを押すと選択項目が移動し、記録（MEM）スイッチを押すと選択中のモードを決定する。

呼出側の状態遷移は次のとおりである。

- `MEASMODE_1` では、電池残量を更新したうえで選択画面（`flg == 0`）を表示する。選択中の値には `measure_mode_flag` を渡す。表示に成功すると点滅フラグを反転させ、表示更新用タイマーを設定して `MEASMODE_2` へ遷移する。
- `MEASMODE_3` では、設定確定後のメッセージ画面を表示する。このとき `flg` には `measure_mode_flag` に 1 を加えた値を渡すため、選択したモードに対応するメッセージが表示される。表示に成功すると、LCD 自動消灯用タイマーと点滅用タイマーを設定して `MEASMODE_4` へ遷移する。

選択画面上で記録（MEM）スイッチにより決定してから `MEASMODE_3` へ至る間の処理を行う状態（`MEASMODE_2`）は本スライスに含まれないため、決定操作から確定メッセージ表示へ至る具体的な遷移は要確認である。

なお、LCD 表示バッファの初期化（`lcd_init`）に失敗した場合は、描画を行わず異常終了する。

## 5.5 MLSS測定画面（待機・測定中）

<sub>実装: 描画 `disp_MLSS_Meas_2`, `disp_MLSS_Meas_3` / 遷移 呼出側 case</sub>

MLSS 測定画面は、プローブから受信した測定値の安定を待つ測定中（安定待ち）画面（`disp_MLSS_Meas_2`）と、安定確定後に測定値を保持して記録操作を促す測定ホールド画面（`disp_MLSS_Meas_3`）の二つで構成される。いずれもモノクロ LCD 400×240 に描画し、`SPI_transmitLCD` で一括転送する。冒頭で `lcd_init` により表示バッファのクリアと更新可否を確認し、更新できない場合は `DISP_NG` を返して当該周期の描画を行わない。

両画面の共通要素として、電池残量アイコン（`LS027_disp_battery_icon`）と日付時刻（`LS027_disp_datetime`）を表示する。画面下段には測定量の種別（`Meas_Mode`）に応じて主表示欄を切り替え、共通ラベルとして「モード」アイコン・「/」・「30」（`icon_mode` ＋ `s_slash` ＋ 数値 30）を並べてモード番号の分母を示す。画面左下には `DISP` アイコンと「切替」アイコン（`icon_DISP` ＋ `icon_change`）を表示し、履歴 DISP スイッチによる測定量切替操作の案内とする。

測定中画面（`disp_MLSS_Meas_2`）では、画面上部に測定中を示す矢印アイコンを表示する。通常は `flash_flg` の値に応じて `icon_wait1` と `icon_wait2` を交互に描画して約 1Hz でアニメーションさせるが、透視度モード（`MEAS_MODE_TR`）で透視度が 200cm 相当以上のときは、レンジオーバーを示す固定アイコン（`icon_over`）へ切り替える。さらに進捗バー（`LS027_disp_bar_icon`）で測定値の安定進捗を表示し、この進捗値は呼出側で `get_stable_prog` により求めた値が渡される。

プローブが無応答または切断され測定値ストリームの鮮度が切れた場合（`Probe_Data_Valid` が偽）は、主表示欄を「----」（`disp_meas_no_data`）に置き換え、「モード / 30」ラベル・進捗バー・`DISP`「切替」ナビゲーションのみを描画して終了する。この無応答表示は測定中画面に限り、測定ホールド画面（`disp_MLSS_Meas_3`）では捕捉済みの結果を消さないため適用しない。

主表示欄は `Meas_Mode` により切り替わる。MLSS モード（`MEAS_MODE_MLSS`）では、タイトルに「MLSS」（`t_MLSS`）を表示し、測定値を表示単位で 10 の位に丸めて描画する。10000 mg/L 以上は g/L 形式（例: 10000→10.0、20000→20.0、分解能 0.1 g/L）で表示して単位「g/L」（`l_gl`）を添え、1000〜9999 mg/L はカンマ区切りの 4 桁で描画して単位「mg/L」（`l_mgl`）を添え、1000 mg/L 未満は整数で表示する。測定値が `FLT_MAX`・20000 以上・-99 以下の範囲外のときは、`flash_flg` に同期して主表示欄を白塗りし点滅させる。MLSS モードではさらに界面判断バー（`LS027_disp_airbar3`）を表示し、現在水深（小サイズ）と界面深度のホールド値（中サイズ、単位「m」）を併記する。界面深度のホールド値が未捕捉のときは「-.--」を点滅させずに定常表示する。

SS モード（`MEAS_MODE_SS`）では、タイトルに「SS」（`t_SS`）、測定値を整数で mg/L 表示する。値が `FLT_MAX`・1000 以上・-99 以下の範囲外のときは白塗り点滅させる。透視度モード（`MEAS_MODE_TR`）では、タイトルに「透視度」（`t_Transparency`）、測定値を小数第 1 位まで cm 表示する。200cm を超える場合は点滅させず 200.0 のまま表示し、レンジオーバーは前述の矢印アイコン（`icon_over`）で通知する。下限（-9.9 以下）および `FLT_MAX` のときのみ白塗り点滅させる。界面モード（`MEAS_MODE_DEPTH`）は水深と MLSS を併記する分岐を持つが、コード上のコメントに「UI 到達不能」と明記されており通常操作からは到達しない。

測定ホールド画面（`disp_MLSS_Meas_3`）は測定中画面と同じ主表示レイアウトを持つが、測定中を示す矢印アイコン・進捗バー・プローブ無応答処理は持たない。代わりに、記録操作の状態を示す `save` 引数に応じて画面左下のナビゲーションを切り替える。`save` が 0 のときは `DISP`「切替」に加えて記録 MEM スイッチ用に「記録」案内（`icon_MEM` ＋ `icon_save`）を表示する。`save` が 1 のときは記録の完了を示す表示（`icon_save` ＋ `icon_complete`）へ、それ以外のときは解除の完了を示す表示（`icon_release` ＋ `icon_complete`）へ切り替える。

呼出側では、測定中画面は表示周期タイマ（`disp_timer` に `DISP_CYCLE`）が満了するたびに `disp_MLSS_Meas_2` を呼び、点滅フラグ（`WAG_flash_flag`）を反転させながら安定待ちを続ける。測定ホールド画面は、安定確定後の待機表示（`save`＝0）と記録完了表示（`save`＝1）の二経路から呼ばれ、記録完了表示では画面を約 2 秒保持したうえで消灯タイマ（`lcd_off_timer`）を設定して次状態へ遷移する。安定成立や各スイッチ押下による具体的な画面遷移条件は本節に渡されたコード片からは読み取れない（要確認）。

## 5.6 MLSS測定完了画面

<sub>実装: 描画 `disp_MLSS_Meas_4` / 遷移 呼出側 case</sub>

MLSS 測定完了画面（`disp_MLSS_Meas_4`）は、測定モードから通常測定モードへの切替を案内するメッセージ画面である。モノクロ LCD 400×240 に描画し、冒頭で `lcd_init` により表示バッファのクリアと更新可否を確認し、更新できない場合は `DISP_NG` を返してその周期の描画を行わない。

画面左上のタイトルは、淡水／海水の設定（`tansui_sw_flag`）に応じて切り替わる。海水設定のときは海水用タイトル（`t_logger_s`）を、淡水設定のときは淡水用タイトル（`t_logger`）を表示する。画面右上には電池残量アイコン（`LS027_disp_battery_icon`）と日付時刻（`LS027_disp_datetime`）を表示する。画面中央には測定モード切替を促すメッセージ（`msg_change_normalmode`）を表示する。全要素を描画したのち `SPI_transmitLCD` で LCD へ一括転送し、`DISP_OK` を返す。

呼出側では、通常測定切替メッセージの表示状態でこの関数を呼び、電池残量の点滅計算（`flash_calc` に `FLASH_BAT`）を行ったうえで描画する。描画が成功すると消灯タイマ（`lcd_off_timer`）と点滅タイマ（`flash_timer` に `DISP_CYCLE`）を設定して次状態へ遷移する。メッセージ表示中のスイッチ操作による具体的な遷移条件は本節に渡されたコード片からは読み取れない（要確認）。



## 5.7 測定レンジ表示

<sub>実装: 描画 `disp_RANGE1`, `disp_RANGE2` / 遷移 呼出側 case</sub>

測定レンジ表示は、感度調整モードにおいてレンジごとの測定電圧を表示する画面群であり、調整待機中を示す画面（`disp_RANGE1`）と調整値を確定して保持する完了ホールド画面（`disp_RANGE2`）の二つで構成される。いずれもモノクロ LCD 400×240 に描画し、冒頭で `lcd_init` により表示バッファのクリアと更新可否を確認し、更新できない場合は `DISP_NG` を返してその周期の描画を行わない。

両画面ともタイトルは対象レンジ（`op_mode`）に応じて切り替わる。低レンジ・中レンジ・高レンジの各感度調整（`LOWRANGE`／`MIDRANGE`／`HIRANGE`）ではそれぞれ `t_range_low`・`t_range_middle`・`t_range_high` を、低・中・高レンジのゼロ調整（`LOWRANGE0`／`MIDRANGE0`／`HIRANGE0`）ではそれぞれ `t_range0_low`・`t_range0_middle`・`t_range0_high` をタイトルに表示する。いずれにも該当しないときはタイトル領域を黒塗り（`lcd_fill_box`）する。画面中央には測定電圧値（`range_mv`）を整数で表示し、単位「mV」（`m_mVoltage`）を添える。

調整待機中画面（`disp_RANGE1`）では、画面左下にナビゲーションとして `DISP` アイコンと「次へ」（`icon_DISP` ＋ `icon_next`）、`MEM` アイコンと「開始」（`icon_MEM` ＋ `icon_start`）を表示する。これは、履歴 DISP スイッチで次のレンジへ進み、記録 MEM スイッチで当該レンジの調整を開始する操作の案内である。表示電圧には呼出側で取得した AD 値（`AD`）が渡される。

調整値ホールド画面（`disp_RANGE2`）は同じタイトルと電圧値・単位を表示するが、ナビゲーションアイコンは持たず、確定した調整値のみを保持表示する。表示電圧には確定した閾値（`domv_thr`）が渡される。

呼出側では、調整待機中画面は調整計算（`adj_calc`）で表示値を更新したうえで `disp_RANGE1` を呼び、対象レンジの上位ビットを保ったまま次の状態へ遷移する。ホールド画面は調整値ホールド表示状態から `disp_RANGE2` を呼び、ホールド表示タイマ（`disp_hold_timer` に `HOLD_DISP`）を設定して次状態へ遷移する。各スイッチ押下による具体的な画面遷移条件は本節に渡されたコード片からは読み取れない（要確認）。

## 5.9 界面閾値設定

<sub>実装: 描画 `disp_Depth_Setting` / 遷移 呼出側 case</sub>

画面「界面閾値設定」は、界面計測における界面閾値（Interface_Threshold）の設定値を桁ごとに入力・変更するための画面である。画面上段に閾値の設定値を大サイズの数字で表示し、下段のスイッチ操作で編集する。以下に表示要素および状態遷移について記述する。

- 電池残量アイコン
- 日付時刻表示
- 界面閾値の設定値。整数5桁を大サイズ数字（l_num）で上段に表示する。編集対象として選択されている桁は、点滅タイミングに合わせて反転表示（b_l_num）となる。
- 単位「mg/L」
- DISPスイッチアイコン（上段・中段の2段）
- MEMスイッチアイコン（下段）

呼出側では、本画面の表示状態から入力処理状態へ移行したうえで本描画関数を呼び出しており、設定中の界面閾値の値と選択中の桁位置を渡して表示している。表示値は界面閾値を整数5桁に分解し、各桁を個別に描画する。

以上が、画面「界面閾値設定」の動作仕様である。



## 6.1 校正メニュー

<sub>実装: 描画 `disp_C_MENU` / 遷移 呼出側 case</sub>

画面「校正メニュー」は、実施する校正の種類を選択するための画面である。校正メニューへ入った経路（電源ON経由か測定メニュー経由か）と、現在の校正モードおよび相関式によって、表示される選択肢が変化する。以下に表示要素および状態遷移について記述する。

表示要素は次のとおりである。

- タイトルアイコン「校正メニュー」
- 電池残量アイコン
- 日付時刻表示

選択肢の内容は経路と状態に応じて次のように切り替わる。

- 電源ON経由で校正メニューに入った場合は、全校正モード共通の配置として「ゼロ校正」「2点校正」「3点校正」「校正リセット」の4項目を表示する。この経路では「測定」への復帰項目は表示しない。

  - 校正モードがゼロ校正（CAL_MODE_Z）のときは「校正リセット」と「測定」を表示する。加えて、現在の相関式が校正可能な場合（内部値が下限インデックス以上の場合）にのみ「2点校正」「3点校正」を表示する。校正不可な相関式（内部値が20未満）では2点校正・3点校正のアイコンを描画しない。また界面（DEPTH）モードは相関式に依らず2点校正・3点校正を非表示とする。
  - 校正モードが2点校正（CAL_MODE_2P）のときは「ゼロ校正」「3点校正」「校正リセット」「測定」を表示する。
  - 校正モードが3点校正（CAL_MODE_3P）のときは「ゼロ校正」「2点校正」「校正リセット」「測定」を表示する。

スイッチ操作の効果は、描画されるスイッチアイコンの意味に従い次のとおりである。

- DISPスイッチを押すと、選択項目が移動する（選択）。
- DISPスイッチを長押しすると、前の画面へ戻る。
- MEMスイッチを押すと、選択中の項目を決定する（決定）。

DISP長押しやMEM決定による具体的な遷移先画面名は要確認（呼出側の入力処理状態のコードが本執筆時点で参照範囲になく、遷移先を確定できない）。呼出側では、本画面の表示状態から入力処理状態へ移行したうえで本描画関数を呼び出しており、校正モード・選択位置・電池残量・点滅状態を渡して表示している。

以上が、画面「校正メニュー」の動作仕様である。

## 6.3 補正設定

- DISPスイッチアイコン（上段・中段の2段）

## 6.4 MLSSゼロ校正

<sub>実装: 描画 `disp_MLSS_ZCal_2`, `disp_MLSS_ZCal_3` / 遷移 呼出側 case</sub>

画面「MLSSゼロ校正」は、測定値を基準ゼロに合わせるための校正画面である。校正中を表示する disp_MLSS_ZCal_2 と、校正の完了・保持を表示する disp_MLSS_ZCal_3 の二種類の描画から構成される。

**校正中画面（disp_MLSS_ZCal_2、状態 ZCAL_4）の表示要素**

- ゼロ校正のタイトルアイコン（t_zero_cal）を画面左上に表示する。
- 電池残量アイコンおよび日付時刻を表示する。
- 校正の進行を示す矢印アイコンを表示する。矢印は表示更新のたびに二形（icon_wait1／icon_wait2）を切り替えて点滅する。
- 測定モードに応じて測定値と単位を表示する。MLSS モードおよび界面（DEPTH）モードでは MLSS 値と「mg/L」を表示し、表示範囲は -99 から 20000 とする。SS モードでは SS 値と「mg/L」を表示し、範囲は -99 から 1000 とする。透視度（TR）モードでは透視度を小数第1位まで表示し、単位は「cm」、範囲は -9.9 から 100.0 とする。
- 測定値が無効（FLT_MAX）または範囲外のときは、矢印点滅に合わせて数値部を白塗りし、点滅表示とする。
- 「モード」アイコン、区切りの「/」、および総モード数「30」とともに、現在の測定モード番号を表示する。
- 記録（MEM）スイッチのアイコンと、その操作が「中止」であることを示す中止アイコンを表示する。
- 校正の進捗バーを表示する。

**校正中画面のスイッチ操作と遷移（ZCAL_4）**

- 記録（MEM）スイッチを押す（MEM_sw_check の戻り値が 3 以上）と、ゼロ校正を中止して ZCAL_1 の画面を表示する。
- 校正の制限時間（60 秒）が経過してもセンサーが安定しない場合は、センサー不安定として扱い、エラーメッセージ画面（ERRMSG）を表示する。この際の戻り先はゼロ校正（ZCAL）として保持される。

時分設定（disp_SETHOUR）では、画面上部にタイトル「時刻設定」を表示し、中央に時と分をそれぞれ2桁で、間をコロンで区切って表示する。時の十の位・一の位、分の十の位・一の位のそれぞれに「＋」「－」のアイコンを上下に配置し、画面下部に「中止」「セット」のアイコンを並べる。画面左下にはDISPキーとMEMキーの割り当てラベルを置く。

編集を始める前の画面では、DISPキー側に「次へ」、MEMキー側に「開始」を表示する。この前画面では操作しないまま一定周期が過ぎるごとに時計を読み直し、表示を最新の値に更新する。前画面でDISPキー（次へ）を押すとプログラムバージョン表示画面へ進む。MEMキー（開始）を押すと時分の編集を開始し、編集画面へ移る。

編集画面ではDISPキー側に「選択」、MEMキー側に「決定」を表示し、選択中の項目を反転アイコンで強調しながら点滅させる。

DISPキー（選択）を押すと、選択位置が、時の十の位「＋」→時の十の位「－」→時の一の位「＋」→時の一の位「－」→分の十の位「＋」→分の十の位「－」→分の一の位「＋」→分の一の位「－」→「中止」→「セット」→先頭、の順に一つずつ移る。

時側の「＋」「－」を選んだ状態でMEMキー（決定）を押すと、十の位を選んでいれば時が10、一の位を選んでいれば時が1、増減する。時は0〜23の範囲に収まり、範囲を超える操作をしても上限23・下限0で止まる。

分側の「＋」「－」を選んだ状態でMEMキー（決定）を押すと、十の位を選んでいれば分が10、一の位を選んでいれば分が1、増減する。分は0〜59の範囲に収まり、範囲を超える操作をしても上限59・下限0で止まる。

「中止」を選んだ状態でMEMキー（決定）を押すと、編集内容を書き込まずに時分設定の前画面へ戻る。

「セット」を選んだ状態でMEMキー（決定）を押すと、表示中の年月日時分を時計へ書き込み、時分設定の前画面へ戻る。



## 7.9 水温校正

<sub>実装: 描画 `disp_CALTEMP` / 遷移 呼出側 case</sub>

水温校正は水温1点調整を行う画面であり、調整開始前と調整中の二つの表示状態を持つ。画面上部にはタイトル「水温校正」を表示し、中央に現在の水温値を小数第1位まで数値表示する。

調整開始前の状態では、DISP スイッチ位置に操作アイコンと「次へ」、MEM スイッチ位置に操作アイコンと「開始」を表示する。DISP スイッチを押下すると日付設定（年）画面を表示する。MEM スイッチを押下すると水温1点調整を開始し、調整中の状態へ移る。

調整中の状態では、DISP スイッチ位置に操作アイコンと「選択」、MEM スイッチ位置に操作アイコンと「決定」を表示する。中央付近には「＋」「－」「初期化」「中止」「セット」の各項目を並べて表示し、現在選択中の項目を反転表示（黒地）で点滅させ、他の項目は白地で表示する。

## 8.1 基板調整：プログラムVer表示

<sub>実装: 描画 `disp_PRGVER` / 遷移 呼出側 case</sub>

プログラムVer表示は、基板調整の先頭に位置する画面であり、本体ファームウェアのバージョンを確認するとともに、エラー履歴の消去を行える。画面上部にタイトル「プログラムVer.」を表示し、その下にファームウェアのバージョン番号を小数第2位まで表示する。DISP スイッチ位置には操作アイコンと「次へ」を表示する。

エラー履歴が残っている通常表示では、MEM スイッチ位置に操作アイコンと「長押しで消去」を表示する。MEM スイッチを長押しすると、エラー発生日（年・月・日）とエラー番号を消去し、その結果を EEPROM へ書き込む。書き込みに成功すると、MEM スイッチ位置のアイコンに代わって「消去完了」を表示する。この消去完了表示は一定時間だけ保持され、時間の経過後は自動的に通常のバージョン表示へ戻る。

MEM スイッチを長押しした際に EEPROM への書き込みに失敗した場合は、メーター異常のエラー画面を表示し、そのエラー解除後の復帰先をプログラムVer表示画面とする。

DISP スイッチを押下すると、EEPROM確認画面へ遷移する。



## 8.2 基板調整：EEPROM確認

<sub>実装: 描画 `disp_EEP1`, `disp_EEP2`, `disp_EEP3` / 遷移 呼出側 case</sub>

EEPROM確認は、本体基板の EEPROM に対して書き込みと読み出しの検査を行い、その正否を表示する機能であり、待機画面・検査中画面・結果画面の三つの画面で構成される。いずれの画面でも上部にタイトル「EEPROM」を表示する。

待機画面では、DISP スイッチ位置に操作アイコンと「次へ」、MEM スイッチ位置に操作アイコンと「開始」を表示する。MEM スイッチを押下すると検査を開始し、検査中画面へ遷移する。

検査中画面では、タイトルの下に矢印アイコンと進捗バーを表示する。矢印アイコンは表示更新のたびに左向きと右向きが交互に切り替わり、検査が進行中であることを示す。進捗バーは、0xAA 書き込み・0xAA ベリファイ・0x55 書き込み・0x55 ベリファイ・0x00 書き込み・0x00 ベリファイの全6工程に対する現在の進捗の割合を表す。全ページに対して各工程を順に実行し、いずれかの工程で異常を検出した場合、または全工程を完了した場合に結果画面へ遷移する。

結果画面では、タイトルの下に判定結果として「OK」または「NG」のアイコンと、100 パーセントを示す進捗バーを表示する。全工程が正常であれば「OK」、途中で異常を検出した場合は「NG」を表示する。DISP スイッチ位置に操作アイコンと「次へ」、MEM スイッチ位置に操作アイコンと「開始」を表示する。結果画面の表示に移る際には、各種初期パラメータと初期情報データの設定、および調整値・1WIRE 情報・1WIRE 予備情報の EEPROM 書き込みを行う。

結果画面で MEM スイッチを押下すると、検査を再度開始する。待機画面・検査中画面・結果画面のいずれにおいても、DISP スイッチを押下すると電池電圧確認画面へ遷移する。

## 9.1 ガイダンス表示

<sub>実装: 描画 `disp_GUIDE1`, `disp_GUIDE2`, `disp_GUIDE3`, `disp_GUIDE4`, `disp_GUIDE5` / 遷移 呼出側 case</sub>

ガイダンス表示は、校正や測定に関する案内を利用者に提示する画面群であり、内容の異なる五つのガイダンスから成る。それぞれ、校正の目安、スパン校正方法、ゼロ校正方法、測定方法、保管方法を案内する。各ガイダンスは、対応する状態になったときに表示される。

いずれのガイダンス画面も共通のレイアウトを持ち、画面右上に電池残量アイコン、上部にガイダンスタイトル、その下に注意枠とガイダンスメッセージを表示する。校正の目安・スパン校正方法・ゼロ校正方法・測定方法の各ガイダンスでは、メッセージの下に案内用のアイコンを表示するが、保管方法のガイダンスではこのアイコンは表示しない。DISP スイッチ位置には操作アイコンと「以後は非表示」、MEM スイッチ位置には操作アイコンと確定を表す表示を配置する。確定の表示は、校正の目安・スパン校正方法・ゼロ校正方法・測定方法の各ガイダンスでは「OK」、保管方法のガイダンスでは「終了」となる。

いずれのガイダンス画面でも、表示後は解除待ちの状態となり、電池残量アイコンが一定周期で点滅する。MEM スイッチを押下すると、ガイダンスを解除して発生元の画面へ戻る。DISP スイッチを押下すると、そのガイダンスを以後は表示しない設定にしたうえで、発生元の画面へ戻る。



## 10.1 エラー表示（共通）

<sub>実装: 描画 `disp_ERROR` / 遷移 呼出側 case</sub>

エラー画面は、QRコードを表示するページ1と、対処手順を表示するページ2の2ページで構成し、いずれも共通の描画テンプレートによって描画する。個々のエラーで異なるタイトル・メッセージ・アイコン・ボタン有無は、レイアウト定義テーブル (`error1_table` / `error2_table`) を差し替えることで表現し、描画処理そのものは全エラーで共通である。

ページ1（QRコード表示）は `disp_error_page1` で描画する。冒頭で `lcd_init(0)` により表示バッファを初期化し、失敗した場合は `DISP_NG` を返して何も描画しない。初期化に成功した場合、次の要素を描画して `DISP_OK` を返す。

- 電池残量アイコン（`batvol` が 0〜3 のとき残量段階を表示、0xFF のとき非表示）
- エラータイトル（座標 0,1）
- エラーメッセージ（座標 0,6）
- QRコード枠（座標 29,7）と、その内側にエラー番号に対応した QRコード本体
- エラーアイコン（座標 16,20）
- レイアウトの `show_procedure` が有効なとき、左下に DISP ラベル（座標 0,19）と「手順表示」ボタンアイコン（座標 5,19）
- レイアウトの `show_release` が有効なとき、最下段左に MEM ラベル（座標 0,24）と「解除」ボタンアイコン（座標 5,24）
- レイアウトに追加描画（`extra_draw`）が指定されている場合はそれを実行する

ページ2（対処手順表示）は `disp_error_page2` で描画する。ページ1と同様、`lcd_init(0)` に失敗した場合は `DISP_NG` を返す。成功時は次の要素を描画する。

- 電池残量アイコン（`batvol` の扱いはページ1と同じ）
- エラータイトル（座標 0,1）
- 対処手順メッセージ（座標 16,6）
- レイアウトの `show_qr_disp` が有効なとき、左下に DISP ラベル（座標 0,19）と「QR表示」ボタンアイコン（座標 5,19）
- レイアウトの `show_release` が有効なとき、最下段左に MEM ラベル（座標 0,24）と「解除」ボタンアイコン（座標 5,24）
- 追加描画（`extra_draw`）が指定されている場合はそれを実行する

エラー表示中の操作待ちには、ページ1の解除待ち状態 `ERRWAIT1` と、ページ2の解除待ち状態 `ERRWAIT2` の2つがある。スイッチは `MEM_sw_check()` / `DISP_sw_check()` の戻り値が 3 以上のとき押下とみなす。エラー表示中のスイッチ操作と画面遷移は、全エラーで次の共通ルールに従う。

- ページ1（`ERRWAIT1`）を表示中に記録スイッチ (MEM) を押下すると、エラーを解除して元の画面（復帰先）を表示する。ただし一部のエラーでは MEM を押しても解除しない（各エラーの記述による）。
- ページ1を表示中に履歴スイッチ (DISP) を押下すると、同じエラーのページ2（対処手順表示）を表示する。ただし一部のエラーでは DISP を受け付けず、ページ2へ遷移しない。
- ページ2（`ERRWAIT2`）を表示中に記録スイッチ (MEM) を押下すると、エラーを解除して元の画面を表示する（一部エラーでは解除しない点はページ1と同じ）。
- ページ2を表示中に履歴スイッチ (DISP) を押下すると、同じエラーのページ1（QRコード表示）を表示する。ただし本体故障 (`err_METER`) では DISP を受け付けない。
- 一部のエラーは、表示タイマの経過により、スイッチ操作が無くても自動的に元の画面へ戻る。
- エラー解除待ちの間、電池残量アイコンは約 500ms 周期（`DISP_CYCLE`）で点滅更新される。

ページ1・ページ2で用いるラベル・ボタンアイコンは共通定義（`disp_ui`）としてまとめており、QRコード枠、DISP ラベル、MEM ラベル、「手順表示」「解除」「QR表示」「以後は非表示」の各ボタンアイコンを保持する。

レイアウトテーブルに該当が無いエラー番号に対しては、簡易表示 `disp_ERROR` を用いる。`lcd_init(0)` に失敗した場合は `DISP_NG` を返す。成功時は次を描画する。

- エラー番号のみを数値として表示（座標 44,5、小数点なし）
- 電池残量アイコン
- MEM ラベル（座標 0,23）と「解除」ボタンアイコン（座標 5,23）

この簡易表示が成功すると `ERRWAIT1` もしくは `ERRWAIT2` の解除待ちへ移り、記録スイッチ (MEM) の押下で元の画面へ戻る。



## 10.2 エラー画面 1〜9

<sub>実装: 描画 `disp_ERROR1_1`, `disp_ERROR1_2`, `disp_ERROR2_1`, `disp_ERROR2_2`, `disp_ERROR3_1`, `disp_ERROR3_2`, `disp_ERROR4_1`, `disp_ERROR4_2`, `disp_ERROR5_1`, `disp_ERROR5_2`, `disp_ERROR6_1`, `disp_ERROR6_2`, `disp_ERROR7_1`, `disp_ERROR7_2`, `disp_ERROR8_1`, `disp_ERROR8_2`, `disp_ERROR9_1`, `disp_ERROR9_2` / 遷移 呼出側 case</sub>

## 11.1 省電力測定中画面

<sub>実装: 描画 `disp_ECOMODE` / 遷移 呼出側 case</sub>

省電力測定中画面は、データロガー測定を継続しながら消費電力を抑えるために、LCD を一時的に点灯して測定中である旨だけを短時間表示する画面である。`disp_ECOMODE()` によって描画され、`operation_mode` が `LOG_9` の状態で表示される。

画面には「測定中」を示すメッセージ（`msg_loging`）のみを画面中央付近に表示する。日付時刻・電池残量・測定値などの詳細情報は表示せず、メッセージ 1 点のみの簡素な構成である。描画に先立ち LCD 表示バッファをクリアし、`lcd_init` が失敗した場合は表示を更新せずに終了する。

この画面はロガー測定中の待機状態（`LOG_6`）から自動的に遷移して表示されるものであり、利用者のスイッチ操作によって開くものではない。具体的には、Bluetooth 非接続かつ LCD が点灯しており、次回のロガー記録まで時間的余裕がある場合に省電力へ移行し、本画面を表示する。

本画面ではメッセージを表示したのち、内蔵の省電力タイマー（`ecomode_timer`）が満了すると LCD の電源を切る指令（`lcd_off_cmd`）を出し、ロガー測定中の待機状態（`LOG_6`）へ戻る。すなわち約 1 秒間だけ「測定中」を表示して LCD を消灯し、以降は画面消灯のまま測定を継続する動作となる。LCD が消灯している間もロガーとしての測定・記録そのものは継続する。

本画面には記録スイッチ（MEM）および履歴スイッチ（DISP）に対する処理は設けられておらず、これらのスイッチを操作しても本画面上では画面遷移や動作の変化は生じない。スイッチによる測定停止などの操作は、LCD 点灯後のロガー測定中画面側で受け付ける。



## 11.2 ロガー測定画面

<sub>実装: 描画 `disp_LOG1`, `disp_LOG3`, `disp_LOG4`, `disp_LOG5` / 遷移 呼出側 case</sub>

ロガー測定画面は、一定間隔で溶存酸素（DO）と水温を自動記録するデータロガー機能の一連の画面であり、測定開始前・測定中・測定完了・モード切替案内の各局面を担う。いずれも ID-200T 由来の機能である。画面のタイトルは、`tansui_sw_flag` により淡水用（`t_logger`）と海水用（`t_logger_s`）が切り替わる。共通して電池残量アイコン・日付時刻・開始日時（年／月／日 時：分）・測定間隔（分）・サンプル数（現在値／最大 2000 点）・DO 値（mg/L）・水温（℃）を表示し、Bluetooth 接続中は BT アイコンを表示する。

ロガー測定開始前の待機画面は `disp_LOG1()` で描画され、`operation_mode` が `LOG_2`（描画）および `LOG_3`（操作待ち）の状態に対応する。画面下部には履歴スイッチ（DISP）側に「切替」、記録スイッチ（MEM）側に「開始」の操作案内アイコンを表示する。サンプル数は開始前のため常に 0／2000 点として表示する。この待機状態では、記録スイッチ（MEM）を押すとロガー測定を開始して測定中画面へ遷移する。履歴スイッチ（DISP）を長押しすると通常測定モードへ切り替わる。履歴スイッチ（DISP）を短く押して離すとロガー履歴表示画面へ遷移する。また、あらかじめ設定した開始時刻の予約がある場合は、時刻到来により自動的に測定を開始する。

ロガー測定中画面は `disp_LOG3()` で描画され、`operation_mode` が `LOG_5` の状態に対応する。測定中であることを示すため、待機アニメーションアイコン（`icon_wait1`／`icon_wait2`）を交互に表示する。記録スイッチ（MEM）側には「長押しで停止」の案内を表示する。この画面で記録スイッチ（MEM）を長押しすると、画面中央に停止確認メッセージ（`msg_loger_stop`）を表示し、履歴スイッチ（DISP）側に「戻る」を表示した停止確認状態となる。停止確認状態でさらに記録スイッチ（MEM）を長押しするとロガー測定を終了し、測定完了画面へ遷移する。停止確認状態で履歴スイッチ（DISP）を押すと停止を取り消して測定中画面の表示に戻る。サンプル数が最大値（2000 点）に達した場合も測定は自動的に完了する。停止確認中でない通常の測定中表示では、開始日時・測定間隔・サンプル数を表示する。

ロガー測定完了画面は `disp_LOG4()` で描画され、`operation_mode` が `LOG_A`（描画）および `LOG_B`（操作待ち）の状態に対応する。待機画面と同様に、履歴スイッチ（DISP）側に「切替」、記録スイッチ（MEM）側に「開始」の案内を表示し、記録した総サンプル数を表示する。この画面で記録スイッチ（MEM）を押すと、あらためてロガー測定を開始する。履歴スイッチ（DISP）を長押しすると通常測定モードへ切り替わる。履歴スイッチ（DISP）を短く押して離すとロガー履歴表示画面へ遷移する。

ロガーモード切替案内画面は `disp_LOG5()` で描画され、`operation_mode` が `LOG_C`（描画）および `LOG_D`（操作待ち）の状態に対応する。この画面のタイトルは測定用（`t_measure`／`t_meas_s`）を用い、ロガーモードへ切り替えた旨のメッセージ（`msg_change_logermode`）を表示する。専用アプリ（Reader）のダウンロード案内を表示する設定（`App_DL_disp_flag` が未設定）の場合は、案内メッセージと QR コードを併せて表示し、履歴スイッチ（DISP）側に「以後は非表示」、記録スイッチ（MEM）側に「解除」の案内を表示する。案内表示中の操作待ちでは、記録スイッチ（MEM）を押すと案内を閉じてロガー測定完了画面またはロガー測定待機画面へ遷移する。履歴スイッチ（DISP）を押すと、以後この案内を表示しない設定にしたうえで同様に遷移する。案内を表示しない設定の場合は、スイッチ操作を待たずに一定時間経過で自動的にロガー測定完了画面またはロガー測定待機画面へ遷移する。遷移先が測定完了画面か測定待機画面かは、直前に測定を完了しているか（`log_comp_flag`）によって決まる。



## 11.3 ロガー履歴表示

<sub>実装: 描画 `disp_DISPLOG` / 遷移 呼出側 case</sub>

ロガー履歴表示画面は、フラッシュに記録済みのデータロガー測定の一覧を確認するための画面である。`disp_DISPLOG()` によって描画され、`operation_mode` が `DISPLOG_2`（描画）および `DISPLOG_3`（表示中）の状態に対応する。ID-200T 由来の機能である。

画面上部にはロガー履歴表示用のタイトル（`t_logger_disp`）・電池残量アイコン・日付時刻を表示する。画面下部には履歴スイッチ（DISP）側に「切替」の操作案内アイコンを表示する。記録スイッチ（MEM）側の「次へ」案内はコード上で無効化されており、表示されない。

一覧は 1 画面に 3 件分を上下に並べて表示する。各件について、通し番号（01・02・03 の固定表示）、記録の開始日時（年／月／日 時：分）、測定間隔（分）、およびサンプル数（点）を表示する。表示する 3 件は、あらかじめ新しい順に並べ替えたインデックス（`log_sort_index[0]`〜`[2]`）で指定された履歴ヘッダ（`log_head`）から読み出す。測定間隔が 0 の履歴についてはサンプル数を 0 点として表示する。

# セクションB：プローブ

> プローブは画面を持たず、本体からRS-232Cで制御される。記述は `protocol-rs232c.md` に基づく。


## B-1 プローブ概要・役割

IM-110T プローブ基板は、水質計測用のアナログ信号取得を担うプローブモジュールである。MLSS（混合液懸濁固形物）や透視度などの水質計測に用いるアナログ信号を取得することを役割とする。

プローブ基板は操作スイッチや表示器を持たない。搭載されているスイッチは `NRST2` ラインに接続されたリセットスイッチ `S1` のみであり、操作ボタンや電源ボタンは備えていない。したがってプローブが単独で計測を開始・表示することはなく、本体基板（IM-110T 本体基板）からの制御に従って動作する。

本体とプローブは RS-232C ケーブルで接続され、本体側がマスターとしてプローブを制御する。コマンドはすべて本体からプローブへ発行され、プローブは要求された測定値や応答を返す。プローブが自発的に送信するのは、測定データ自動送信（MS）が有効なときの周期送信のみである。

プローブはバッテリーを搭載していないため、単独では動作できない。電源は本体基板の CN4 コネクタからケーブル経由で供給される 5V（`VIN`）のみであり、ケーブルを抜くとプローブの電源は断たれる。プローブ単独でデバッグを行う場合は、外部から +5V を供給する必要がある。


## B-2 ハードウェア構成（電源・MCU）

プローブ基板の MCU は STM32G070KBT6 である。LQFP32 パッケージの Cortex-M0+ コアであり、最大動作周波数は 64MHz、Flash は 128kB、SRAM は 36kB である。

本体基板の MCU は STM32L452RET6（Cortex-M4）であり、プローブとは MCU 系列が異なる。HAL ドライバ（`STM32G0xx_HAL_Driver` と `STM32L4xx_HAL_Driver`）、ピン配置、ペリフェラルの互換性はいずれも別物であるため、片側のコードをそのまま他方に持ち込むことはできない。

電源構成は以下のとおりである。

- 本体基板の CN4 からケーブル経由で `VIN`（5V）が供給される。プローブにはバッテリーが無く、単独動作はできない。
- `VIN` は Q1（RQ5A030AP、P-MOSFET）による電源逆接保護を経て `+05P` レールとなる。
- `+05P` から MAX8881EUT33-T（IC14）を経由して `+3.3V` が生成され、FB3 フェライトビーズを通してアナログ用の `+3.3VA` となる。
- `+05P` は LM27761DSGT（IC15）インバーティング・チャージポンプによってアナログ用の ±5V（`+05A` / `-05A`）を生成する。
- LM336BMX-2.5（IC9）と 2SC2712（IC8）により +2.5V 基準電圧を生成し、R39/R40 の分圧と C49 で `1V_REF` を生成する。

なお、本体基板が搭載する LTC3221 チャージポンプはプローブ基板には非搭載である。


## B-3 計測フロントエンド（アナログ信号系）

プローブ基板の計測フロントエンド（アナログ計測系）は、オペアンプ、アナログマルチプレクサ、外部 ADC で構成される。

オペアンプは TSX562IST を 7 個（`IC1` から `IC7`）搭載している。低オフセット・CMOS 入力のレール to レール オペアンプである。

アナログマルチプレクサは TMUX4053BQBR を 2 個（`IC10`、`IC11`）搭載している。3 回路 2:1 チャネルのアナログスイッチであり、`SEL` / `SEL3` ネット（STM32 の `IC20.13` / `IC20.14`）によってチャネルを切り替える。

外部 ADC は MCP3424T-E/SL（`IC19`）を使用する。4ch・18bit の ΔΣ ADC であり、I2C1（STM32 の `IC20.1`=SDA / `IC20.32`=SCL）で接続される。計測系では STM32 内蔵 ADC は使用しておらず、この外部 ADC で計測を行う。

入力保護として、IC19 の各差動入力には 1N4148WS（`D4`〜`D11`）によるクランプが設けられている。

主要信号系のネットは、回路図ブロック単位の信号名として `MLSS_JUKO` / `MLSS_REF` / `TRA_JUKO` / `TRA_REF` が定義されている。


## B-4 測定チャネルとMD応答

プローブは `MD` コマンドに対し、現在の測定値を 1 回送信する。応答フォーマットは次のとおりである。

```
<mV[0]>, <mV[1]>, <mV[2]>, <mV[3]>, <mV[4]>, <pressure hPa>\r\n
```

6 個の `float` をカンマとスペースで区切って送信する。前半 5 個は内部配列 `AD_AVE_mV[5]`（単位 mV）、6 番目は気圧センサ値である。移動平均は `SADA` コマンドで設定した回数で計算済みの値が入る。この書式は測定データ自動送信（MS 有効時の周期送信）でも同一である。

`AD_AVE_mV[]` の各 index は MCP3424 の物理チャネルと 1 対 1 に対応するのではなく、`SEL3`（`SS` コマンド）の状態によって index が動的に変わる点に注意する。各要素の対応は以下のとおりである。

- `AD_AVE_mV[0]`: MCP3424 CH1（`SADC` の bit0 が 1 のとき更新）
- `AD_AVE_mV[1]`: MCP3424 CH2（`SADC` の bit1 が 1 のとき更新）
- `AD_AVE_mV[2]`: MCP3424 CH3 の SEL3 = LOW 時（`SADC` の bit2 が 1 かつ `SS,0`）。通常ゲイン x1 の測定
- `AD_AVE_mV[3]`: MCP3424 CH4（`SADC` の bit3 が 1 のとき更新）
- `AD_AVE_mV[4]`: MCP3424 CH3 の SEL3 = HIGH 時（`SADC` の bit2 が 1 かつ `SS,1`）。同一の CH3 を PGA gain x2 で再測定した値

すなわち `SS,0` と `SS,1` を切り替えると、同じ物理 CH3 の読み取り値が `[2]` と `[4]` のどちらへ流れるかが切り替わる。更新されなかった側には古い値が残留する。一度も有効化されていないチャネルはゼロ初期化により `0.000000` であり、一度有効化された後に無効化されたチャネルは直前の値が残留する。

6 番目の気圧（pressure hPa）は、HSPPAD143A 気圧センサー（I2C2 / PA11=SCL, PA12=SDA）の実測値である。現行リビジョンでは実装済み・稼働中であり、周期取得した hPa を返す。`MD` 応答書式は固定であり、6 番目の float に実測 hPa が入る。なお HSPPAD143A 内蔵の温度センサ値は `MD` 応答には採用しない方針である。

## B-6 ゼロ点・スパン補正（SADZ/SADS）

アナログ計測値の正規化と調整に関わるコマンドとして、SADZ・SADS・SADA・SADC の4種がある。いずれも指定に対して OK / NG を返す。現行の AD 正規化はゼロ減算を廃し、span（傾き）一本で行う方式である。

- SADZ（ゼロ点）: 2026-07-21 の方式変更でオフセット（ADZero）を撤去したため、現在は計算に使用しない廃止扱いのコマンドである。過去互換のためコマンド自体は残置しており、受理すると ADC_Zero を設定するが、正規化式では参照しない。副作用（ADC_Zero 上書き）があるため本体側ではリトライ対象外とする。

- SADS（スパン, 空中1700 span 傾き）: MLSS AD0 の能動側コマンドである。指定 ch の空中での生 mV を ADC_Span に記録し、span 正規化「出力 = 生 mV × ADC_SPAN_INI / ADC_Span」により空中出力を 1700mV に合わせる（ADC_SPAN_INI = 1700 のため空中で 1700 となる）。取得ゲートは生 mV 1400〜2048mV であり、範囲外は NG を返す。空中 AD 値は時間経過でドリフトするため、本体側ではリトライ対象外とする。

- SADA（移動平均）: ADC の移動平均回数を 0〜255 の範囲で設定する。既定値は 30 である。ノイズ抑制のための平均段数を決める。

- SADC（有効 ch マスク）: 4文字の '0'/'1' で MCP3424 の CH1〜CH4 の有効・無効を指定する。最左文字が bit3（ch3）、最右文字が bit0（ch0）に対応する。有効化した ch のみ AD_AVE_mV[] が更新され、無効化した ch は直前値が残留する。

これらの設定はまず RAM 上の live パラメータへ反映される。恒久化する場合は WPP を発行して内蔵 Flash に保存し、次回起動時に自動で読み込ませる。ただし校正係数・調整パラメータの本命の保管先は統合ストア（probe_store_t, 16ページ×32B = 512B）であり、ADC_Span 等は共通調整ページ（Page1）に配置される。統合ストアへの保存は、変更したページを WPG で送ってから WSC で封緘・flash 確定する流れで行う。運用中の校正・調整は本体 RAM に反映し、電源 OFF の一括書戻し窓でプローブ（WPG×変更ページ＋WSC）と本体 EEPROM ミラーの双方へ、同一 ID・同一最終更新日で書き込む。


## B-7 補正式・調整パラメータ

プローブが取得した生 mV から物理量（MLSS / SS / 透視度）への換算、すなわち濃度換算はすべて本体側で実行する。プローブが担うのは、生 AD 値の span 正規化（出力 = 生 mV × ADC_SPAN_INI / ADC_Span, ADC_SPAN_INI = 1700）と気圧の取得までであり、物理量への補正式そのものはプローブのアプリケーションでは展開しない。

補正に用いる全係数と調整パラメータは、プローブ側 flash の統合ストア（probe_store_t）を唯一の真実源として保管する。本体側 EEPROM はこのストアとバイト完全同一のミラー（非権威キャッシュ）であり、本体基板のみの交換でもプローブの付け替えでも測定値が正しく出るよう、3層モデルで整合を保つ。係数・パラメータの本体⇔プローブ転送は 32B ページ単位で行い、読み出しは RPG、ページ単位の書き込みは WPG、封緘・flash 確定は WSC を用いる。per-field の float テキスト化はせず、共有 struct の生バイト列をそのまま転送するため往復誤差は生じない。

調整パラメータは物理量換算式とは別レイヤに位置づけられ、測定フロントエンドの動作点を決めるものである。代表例として、LED duty（SP コマンド）は LED_REFV の PWM duty を設定して AD 入力レンジを整え、移動平均回数（SADA）はノイズ抑制の平均段数を、有効 ch マスク（SADC）は更新対象 ch を決める。これらは係数（濃度換算式のパラメータ）とは区別して扱う。

具体的な補正式（出荷時2次式、TR 累乗、ゼロ・温度補正、k_No 等）およびページ意味論・校正手順の詳細は、付録 A と校正手順（mlss-calc-reference.md §8 / §12、統合ストア各ページ定義）を参照すること。本節では個々の計算式は展開しない。


## B-8 エラー処理・タイムアウト

プローブは、解釈できないコマンド、およびパラメータの欠落・形式不正・範囲外を受け取った場合に `NG\r\n` を返送する。NG には理由コードを付さないため、原因の特定は本体側が直前に送信したコマンドとの照合で行う。

コマンド応答のタイムアウトとリトライは本体側で管理する。タイムアウトは fast 500ms と slow 3000ms の2区分であり、RPP・WPP・RPF・RPG・WSC のように flash 操作や複数行ダンプを伴うコマンドは slow に分類される。応答タイムアウト時は原則1回リトライするが、SADS・SADZ（AD 値ドリフトの副作用）および MS の引数省略形（トグルの副作用）はリトライ対象外とする。

flash 書込中（WPP / WSC / RPF 等）の CPU ストールで発生し得るオーバーラン（ORE）は、プローブ側の `HAL_UART_ErrorCallback()`（Ver.0.17 で実装済）が救済する。PE / FE / NE / ORE の各フラグを浄化したうえで DMA 受信を再アームすることで、ORE による受信停止から復旧し、以後のコマンド・測定値ストリームの取りこぼしを防ぐ。

タイミング（§4）およびエラー処理（§5）の詳細は docs/protocol-rs232c.md を参照すること。

# 付録


## 付録A 計算式・補正式

本付録は、IM-110 が水質計測値（MLSS・SS・透視度）および界面（水深）を算出する際に用いる計算式・補正式を、現行ファームウェア実装（本体 `IM-110/Core/Src/IM_110.c`、プローブ `IM-110_Probe/Core/Src/IM_110.c`）を真実源として記載したものである。各式が MLSS・SS・透視度・界面のいずれに属するかを明示する。数式中の乗算は `*`、除算は `/`、括弧は `()` で表記する。

計算チェーン全体は、プローブ側で生 AD 値を mV に正規化して送信し、本体側でその mV から吸光度 ABS を求め、ベース式（相関式）で物理量へ変換し、最後にスパン校正を適用する流れである。旧仕様にあった受光暗時オフセット（ADZR 減算）および機差補正 kiza は、現行実装では廃止・削除済みである。

（プローブ側：生 mV 正規化 — MLSS・SS・透視度 共通）

プローブは AD 値を mV に正規化して本体へ送信するのみであり、補正・物理量計算は行わない。正規化は原点を通る傾き（span）補正のみで、空中出力が `ADC_SPAN_INI` になるようにする。旧仕様の `ADC_Zero` オフセット減算は廃止された（`ADC_Zero`／`SADZ` はコマンドのみ残置・計算未使用）。

```
プローブ出力mV = 生mV * ADC_SPAN_INI / ADC_Span[ch]
生mV = digit * LSB
```

ch4（透視度）のみ、span 正規化の前に基準オフセット項が入る。

```
生mV_ch4 = digit * LSB_ch4 + ADC_CH4_VREF
プローブ出力mV_ch4 = 生mV_ch4 * ADC_SPAN_INI / ADC_Span[4]
```

- `プローブ出力mV`: 本体へ送信する正規化後の電圧値 (mV)。空中では概ね `ADC_SPAN_INI` になる。
- `生mV`: ADC のデジタル値 `digit` を `LSB` で電圧に変換した生の値 (mV)。
- `digit`: 外部 ADC（MCP3424）のデジタル出力値。
- `LSB`: 1 digit あたりの電圧。ch1〜4（PGA=1）は 0.0625 mV/digit、ch5（ch4, PGA=2）は 0.03125 mV/digit。
- `ADC_Span[ch]`: 各 ch の空中出力（＝空中の生 mV）を記録した傾き基準。設定コマンド `SADS`。初期値 `ADC_SPAN_INI`。
- `ADC_SPAN_INI`: 正規化ターゲット（空中出力の目標 mV）。1700.0（旧 1850／1750 から変更）。
- `ADC_CH4_VREF`: ch4（透視度）IN- の 1V 基準オフセット補正。1000.0 mV。

なお本体へ届く値は移動平均後の `ADC_mV_ave[ch]`（平均回数 `SADA`、既定 30）であり、瞬時値は `ADC_mV[ch]` である。

（本体側：吸光度 ABS の算出 — MLSS）

本体は正規化済みの受光 mV（Y0）と Ref mV から、Ref を温度プロキシとした 20℃換算の分子 `I(0)` を求め、その `I(0)` と Y0 の比の常用対数を吸光度 ABS とする。受光暗時オフセット（Y0 − ADZR）は廃止され、分母は Y0 そのものである。受光ゼロ基準は分子側の `MLSS_ZR`（清水中オフセット）が担う。

```
d = Ref - MLSS_Coef_ADZR[1]
I(0) = MLSS_ZR + d * (MLSS_Coef_TempC[0] + d * MLSS_Coef_TempC[1])
ABS = log10(I(0) / Y0)
```

- `Y0`: MLSS 受光の正規化済み mV（平均値 `ADC_mV_ave[0]`、瞬時値 `ADC_mV[0]`）。0 以下のときは 0.001 にクランプ。
- `Ref`: MLSS Ref の正規化済み mV（`ADC_mV_ave[1]`）。
- `MLSS_Coef_ADZR[1]`（refZR）: Ref 基準（20℃清水 Ref）。温度校正 `ATCF` が設定。未校正は 0。
- `d`: 20℃基準からの Ref 変化幅（`Ref − refZR`）。
- `MLSS_ZR`（MLSSZR）: 清水中ゼロ基準（分子の基準 mV）。初期値 `INI_MLSS_ZR = 1700.0`、清水ゼロ校正 `AZC` が実値で上書き。
- `MLSS_Coef_TempC[0]`（B）／`MLSS_Coef_TempC[1]`（B2）: Ref 温度補正の 1 次・2 次係数。未校正は 0。
- `I(0)`: 温度補正込みの分子。`I(0) ≤ 0` のときは安全のため `MLSS_ZR` に置換。
- `ABS`: 吸光度。温度補正が未校正（B=B2=0）なら `d*(…)=0` となり `I(0)=MLSS_ZR` に帰着する。清水中では `MLSS_ZR=Y0` となり `ABS=0`（MLSS=0）となる。

（本体側：吸光度 ABS の算出 — SS）

SS も MLSS と同一構造である。受光は `ADC_mV_ave[2]`、Ref は `ADC_mV_ave[3]` を用いる。

```
d = Ref - SS_Coef_ADZR[1]
I(0) = SS_ZR + d * (SS_Coef_TempC[0] + d * SS_Coef_TempC[1])
ABS = log10(I(0) / Y0)
```

- `Y0`: SS 受光の正規化済み mV（`ADC_mV_ave[2]`）。0 以下は 0.001 にクランプ。
- `Ref`: SS Ref の正規化済み mV（`ADC_mV_ave[3]`）。
- `SS_Coef_ADZR[1]`（refZR）: SS の Ref 基準（20℃清水 Ref）。
- `SS_ZR`（SSZR）: SS 清水中ゼロ基準。初期値 `INI_SS_ZR = 1700.0`。
- `SS_Coef_TempC[0]`（B）／`SS_Coef_TempC[1]`（B2）: SS の Ref 温度補正 1 次・2 次係数。
- `I(0)`: SS の温度補正込み分子。`I(0) ≤ 0` は `SS_ZR` に置換。

（本体側：吸光度 ABS の算出 — 透視度）

透視度は SS の Ref（`ADC_mV_ave[3]`）および SS の温度補正係数を流用する。分子の温度補正は 1 次項（B）のみで、2 次項は用いない。

```
d = Ref - SS_Coef_ADZR[1]
I(0) = TR_ZR + d * SS_Coef_TempC[0]
ABS = log10(I(0) / Y0)
```

- `Y0`: 透視度 受光の正規化済み mV（`ADC_mV_ave[4]`）。0 以下は 0.001 にクランプ。
- `Ref`: SS と共有する Ref mV（`ADC_mV_ave[3]`）。
- `SS_Coef_ADZR[1]`（refZR）: SS から流用する Ref 基準。
- `TR_ZR`（透視度ZR）: 透視度 清水中ゼロ基準。初期値 `INI_TR_ZR = 1700.0`。
- `SS_Coef_TempC[0]`（B）: SS から流用する温度補正 1 次係数。
- `I(0)`: 透視度の分子。`I(0) ≤ 0` は `TR_ZR` に置換。

（ベース式：ABS → 物理量 — MLSS）

MLSS は選択中の相関式 No. によりベース値（FABSS）を求める。No.1〜20 は共通の基準 2 次式（No.1 のテーブル `MLSS_Mode_CF[0]`）を傾きゲイン `k_No` で倍する。No.21〜30 は各スロット独立の 2 次式（ゲインは 1.00 固定）を用いる。基準式の評価 `eval_modecf` は係数枠 `modecf[7]`（最大 6 次）を持つが、現行の出荷時ベースは実質 2 次（3 次以上は 0）である。

```
eval_modecf(c, ABS) = c[0] + c[1]*ABS + c[2]*ABS^2 + c[3]*ABS^3 + c[4]*ABS^4 + c[5]*ABS^5 + c[6]*ABS^6
FABSS(No.1〜20) = eval_modecf(MLSS_Mode_CF[0], ABS) * k_No_MLSS[idx]
FABSS(No.21〜30) = eval_modecf(MLSS_Mode_CF[idx], ABS)
```

- `idx`: 選択中相関式のインデックス（`MLSS_MODE`）。0〜19 が No.1〜20、20〜29 が No.21〜30。
- `MLSS_Mode_CF[0]`: No.1 基準 2 次式の係数（No.1〜20 のベースカーブ）。
- `MLSS_Mode_CF[idx]`: No.21〜30 各スロットの独立 2 次式係数。
- `k_No_MLSS[idx]`: No.1〜20 の傾きゲイン。No.1=1.0000、以降等間隔（step ≈ 0.099364）で増加し No.20=2.8879。具体値は `{1.0000, 1.0994, 1.1987, 1.2981, 1.3975, 1.4968, 1.5962, 1.6955, 1.7949, 1.8943, 1.9936, 2.0930, 2.1924, 2.2917, 2.3911, 2.4905, 2.5898, 2.6892, 2.7886, 2.8879}`。
- `FABSS`: スパン校正適用前のベース MLSS 値（`MLSS_FABSS`）。

（ベース式：ABS → 物理量 — SS）

SS は相関式 No. を持たず、校正可能な単一式（No.21 相当、`SS_Mode_CF[SS_MODE]`、`SS_MODE` 初期値 20）のみを用いる。ゲイン倍は行わない。

```
FABSS = eval_modecf(SS_Mode_CF[SS_MODE], ABS)
```

- `SS_Mode_CF[SS_MODE]`: SS の単一 2 次式係数。
- `FABSS`: スパン校正適用前のベース SS 値（`SS_FABSS`）。

（ベース式：ABS → 物理量 — 透視度）

透視度の出荷時ベースは累乗式であり、係数 `a`（`TR_Coef_ModeCF[0]`）と `b`（`TR_Coef_ModeCF[1]`）で表す。`a=0`（未校正）の場合のみ旧多項式 `TR_Mode_CF[TR_MODE]` へフォールバックする。

```
FABSS = a * ABS^b            (a != 0 のとき)
FABSS = eval_modecf(TR_Mode_CF[TR_MODE], ABS)   (a == 0 のとき)
```

- `a`（`TR_Coef_ModeCF[0]`）: 累乗式の係数。
- `b`（`TR_Coef_ModeCF[1]`）: 累乗式の指数。
- `ABS`: 透視度の吸光度。`powf(負, 非整数)=NaN` を避けるため、`ABS < 1e-4` のときは 1e-4 にクランプする。
- `FABSS`: スパン校正適用前のベース透視度 [cm]（`TR_FABSS`）。

（スパン校正：FABSS → 最終出力 — MLSS・SS・透視度 共通形）

ベース値 FABSS に対し、現場 2 点／3 点校正で作成した 2 次のスパン校正式を適用して最終出力を得る。3 モードとも同一形（`A*f^2 + B*f + C`）である。

```
MLSS = MLSS_SP_A * FABSS^2 + MLSS_SP_B * FABSS + MLSS_SP_C
SS = SS_SP_A * FABSS^2 + SS_SP_B * FABSS + SS_SP_C
透視度 = TR_SP_A * FABSS^2 + TR_SP_B * FABSS + TR_SP_C
```

- `*_SP_A`／`*_SP_B`／`*_SP_C`: 各モードのスパン校正係数（2 次・1 次・定数）。初期値は `SP_A=0.0`、`SP_B=1.0`、`SP_C=0.0` であり、未校正時は出力 = FABSS となる。
- `FABSS`: 各モードのベース値（`MLSS_FABSS`／`SS_FABSS`／`TR_FABSS`）。
- `MLSS`／`SS`／`透視度`: 表示される最終物理量。透視度は cm 単位。

MLSS の界面バー表示に用いる瞬時値 `MLSS_inst` も同式で、瞬時 mV（`ADC_mV[0]`／`ADC_mV[1]`）から求めた瞬時 FABSS に対して算出する。

```
MLSS_inst = MLSS_SP_A * FABSS_inst^2 + MLSS_SP_B * FABSS_inst + MLSS_SP_C
```

- `FABSS_inst`: 瞬時受光・瞬時 Ref から求めたベース MLSS 値。
- `MLSS_inst`: 界面判断バー用の瞬時 MLSS 値（MLSS・界面）。

（界面：水深換算 — 界面）

水深は、電源 ON 直後の大気圧を原点（自動ゼロ）とした相対気圧に傾き `Depth_k` を掛けて算出する。オフセットは自動ゼロのため、校正は傾きのみの 1 点スロープ校正である。

```
Depth = (P - Depth_offset) * Depth_k
```

- `P`: 現在の圧力値（プローブ気圧センサ由来、本体 `ADC_mV[5]`）。
- `Depth_offset`: 電源 ON 直後の初回計測で取得した大気圧（原点）。校正画面での界面ゼロ校正でも現在値へ上書き可能。
- `Depth_k`: 水深換算の傾き [m/hPa 相当]。初期値 `1.0 / 100.0`（現行 100hPa/m 相当）。
- `Depth`: 算出された水深 [m]。負値は 0.0 にクランプする。

傾きの 1 点スロープ校正（6m 相当の加圧時に 6.0m 表示となるよう調整）は、電源 ON 大気圧を原点とした相対気圧 `dp` に対して次式で `Depth_k` を更新する（`dp ≤ 0` のときは校正失敗）。

```
dp = P - Depth_offset
Depth_k = 6.0 / dp
```

- `dp`: 電源 ON 大気圧を原点とした相対気圧。
- `Depth_k`: 6m 相当加圧時に `Depth=6.0` となるよう更新される傾き（界面）。


## 付録B EEPROM割付表

本付録は、IM-110 本体基板が搭載する EEPROM（M95256、容量 32KB、SPI 接続）のページ割付を示すものである。運用単位は 1 ページ = 32 バイト（`EEP_PAGE_BYTE = 32`）であり、各ページの末尾 1 バイト（オフセット 31）は XOR チェックサムに充てられる。したがって 1 ページで実データとして扱えるのは先頭 31 バイトである。

ハードウェア上は 1024 ページが存在するが、実運用で使用する範囲は page 0 から page 125 までであり、`EEP_PAGE_MAX = 126` で範囲を絞り込んでいる。これは製造時における EEPROM 初回クリア時間を短縮するための措置であり、データロガー機能を実装する段階で `EEP_PAGE_MAX` を page 126 以降へ再拡張する想定である。

起動時には page 0 の先頭 `PRO_SIZE` バイトが `PRO_NAME`（"IM-110"）と一致するか検査され、不一致の場合は page 0 から page 125 までを初期値で再構築する。page 126 以降には触れない。

なお、相関式と校正の関係については、MLSS / SS は相関式 No.1-30（内部値 0-29）を持ち、No.1-20 は固定係数で校正不可、No.21-30 のみ校正可能で EEPROM に係数を保存する。TR（透視度）は相関式選択が無く、校正係数は 1 セットのみで page 59-60 を直接読み書きする。


## 付録C 相関式・項目別演算

本付録は、吸光度 ABS から測定物理量へ変換する「相関式」と、測定項目ごとの演算構成を、現行の真実源 (docs/mlss-calc-reference.md §8) と本体実装 (IM-110/Core/Src/IM_110.c) から整理したものである。数式はすべてプレーンテキストで示し、乗算を *、除算を / で表す。


**相関式の統一形 (No.1〜30 共通)**

MLSS の 1 次演算 (ABS から FABSS への変換) は、選択中の相関式 No. に応じて次の形をとる。実装は MLSS_FABSS_from_raw_mv() 内にある。

```
FABSS = curve(ABS) * k
  No.1〜20 : curve = 出荷時2次式(基準=No.1),  k = k_No (No.1=1.00 … No.20 ≒ 2.8879)
  No.21〜30: curve = 各スロット独立の2次式,    k = 1.00 (固定)
```

ここで curve(ABS) は modecf[7] 枠を持つ多項式 (Σ c[i] * ABS^i, 最大6次) を評価する eval_modecf() で計算される。ただし出荷時ベースは 3 点校正で確定するため実体は 2 次式であり、3 次以上の係数は 0 である。基準カーブ MLSS_Mode_CF[0] は現状 1 次直線 (C1=21315, 他 0) であり、出荷時 3 点調整 (AMC/AMCF) を実施するまでは従来同等の線形出力となる (要確認: 出荷時 2 次式データは未取得)。


**No.1〜20 = 出荷時2次式の傾きゲイン k_No 倍**

No.1〜20 は FW 固定 (校正不可) であり、基準カーブ (No.1 = MLSS_Mode_CF[0]) を共通に用い、傾きをゲイン係数 k_No で倍する。実装は次のとおりである。

```
FABSS = eval_modecf(MLSS_Mode_CF[0], ABS) * k_No_MLSS[No.-1]
```

- 基準 No.1 は k_No = 1.00 であり、No. が上がるにつれ等間隔 (step ≒ 0.099364) で傾きが増大する。
- 実配列 k_No_MLSS[20] (IM_110.c) の値は次のとおりで、これがゲインの真実源である。

| No. | k_No | No. | k_No | No. | k_No | No. | k_No |
|---|---|---|---|---|---|---|---|
| 1 | 1.0000 | 6 | 1.4968 | 11 | 1.9936 | 16 | 2.4905 |
| 2 | 1.0994 | 7 | 1.5962 | 12 | 2.0930 | 17 | 2.5898 |
| 3 | 1.1987 | 8 | 1.6955 | 13 | 2.1924 | 18 | 2.6892 |
| 4 | 1.2981 | 9 | 1.7949 | 14 | 2.2917 | 19 | 2.7886 |
| 5 | 1.3975 | 10 | 1.8943 | 15 | 2.3911 | 20 | 2.8879 |

- k_No は全プローブ共通の固定 progression であるため、フラッシュには保存せず FW 定数表として持つ。保存されるのは「選択中の No.」のみである。


**No.21〜30 = 各スロット独立の2次式**

No.21〜30 は校正可能なスロットであり、それぞれが独立した 2 次式を持つ。ゲイン k は 1.00 固定である (傾きは 2 次式係数に内包されるため追加ゲインを持たない)。実装は次のとおりである。

```
FABSS = eval_modecf(MLSS_Mode_CF[No.-1], ABS)     (No.21〜30, k は掛けない)
```

- 各スロットの係数は、現場の 3 点校正 (AMC で ABS と基準器値の点を捕捉し、AMCF で多項式フィット) により統合ストアの modecf を上書きして作成する。Apply_Coef_To_Live() で live 側 MLSS_Mode_CF へ反映する。
- 未調整 (初期) 状態では、全 10 スロットに出荷時基準 (No.1 = MLSS_Mode_CF[0]) をコピーして持つ。したがって未調整の No.21 を選んでも No.1 (基準 × 1.00) と同結果になる。
- 相関式初期化 (ACRI コマンド) は No.21〜30 を出荷時基準の再コピーへ戻す。実装は MLSS_Mode_CF[i] へ MLSS_Mode_CF[0] を memcpy する。


**FABSS からスパン校正を経て最終値へ**

相関式で得た FABSS の上に、現場の 2 点 / 3 点スパン校正 (SP_A/B/C) が乗り、最終物理量となる。MLSS の場合は次のとおりである。

```
MLSS = MLSS_SP_A * FABSS^2 + MLSS_SP_B * FABSS + MLSS_SP_C
```

- 初期値は SP_A=0.0, SP_B=1.0, SP_C=0.0 であり、未校正時は MLSS = FABSS となる。
- ゼロ校正の受光ゼロ基準 (清水中オフセット MLSS_ZR) は分子側 I(0) に効くもので、スパン校正 (SP) とは別段である。


**SS / 透視度は相関式 No. を持たない (No.21 の単一式のみ)**

SS と透視度 (TR) には No.1〜20 のゲイン選択がなく、単一の校正可能式のみを持つ。単一式ゆえ、校正モードでゼロ校正と 2 点 / 3 点校正が自然に実施できる。

- SS: 実装は eval_modecf(SS_Mode_CF[SS_MODE], ABS) の単一式で、コメントにも「SS は No.21 単一式 (相関式 No.無し)」とある。上に SS_SP_A/B/C の 2 次スパン校正が乗る。
- 透視度 (TR): 出荷時ベースが累乗式 cm0 = a * ABS^b (a=TR_Coef_ModeCF[0], b=[1]) であり、その上に MLSS/SS と同一形の 2 次補正 cm = A * cm0^2 + B * cm0 + C が乗る。a=0 (未校正) の場合のみ旧多項式 eval_modecf(TR_Mode_CF[TR_MODE], ABS) へフォールバックする。TR の Ref 温度補正は SS のもの (SS_Coef_ADZR[1], SS_Coef_TempC[0]) を流用する。

係数の真実源は、いずれもプローブ側フラッシュ (統合ストア) である。補正・計算そのものは本体側で行う (プローブは AD 値 mV を出力するのみ)。


**測定項目別の演算構成**

各測定項目の相関式・ベースカーブ・上に乗る校正・入力 ch の対応を次に示す。

| 測定項目 | 相関式 No. | 出荷時ベースカーブ | 上に乗る校正 | 受光/Ref ch (code index) |
|---|---|---|---|---|
| MLSS | No.1〜20 (k_No ゲイン) + No.21〜30 (独立2次式) | 2次式 (基準=No.1) | スパン SP (2次) | 受光=ch1 [0] / Ref=ch2 [1] |
| SS | No.21 単一式のみ | 2次式 | スパン SP (2次) | 受光=ch3 [2] / Ref=ch4 [3] |
| 透視度 (TR) | No.21 単一式のみ | 累乗式 a * ABS^b | 2次補正 A*cm0^2+B*cm0+C | 受光=ch5 [4] / Ref=ch4 [3] (SS と Ref 共有) |
| 界面 | (MLSS の相関式を流用) | MLSS と同じ | MLSS の SP を流用 | MLSS と同じ (瞬時値 ADC_mV[0]/[1]) |

界面 (界面バー / 界面深度ホールド) は独立した相関式を持たず、MLSS の演算結果を用いる。

- 界面バーには移動平均を掛けない瞬時 MLSS (MLSS_inst) を用いる。算出式は MLSS と同一で、瞬時 AD 値 ADC_mV[0]/[1] から FABSS を求め、MLSS_SP_A/B/C の同じスパン校正を掛ける。
- 界面深度ホールドは、移動平均後の MLSS が Interface_Threshold を下から上へ越えた瞬間に、その時点の水深 Depth を Interface_Hold として捕捉する (Update_Interface_Hold)。したがって界面は「MLSS 値による閾値判定」と「水深の捕捉」の組合せであり、専用の相関式係数は持たない。


**演算チェーン全体 (上流から下流)**

各測定項目に共通する演算の流れは次のとおりである (§0 全体演算チェーン)。

```
プローブ AD (mV, 空中1700 span正規化)
  → 本体: I(0) = ZR + (Ref − refZR)*(B + (Ref − refZR)*B2)   (Ref温度補正, 20℃換算)
  → ABS = log10( I(0) / Y0 )                                  (暗時ADZR減算は廃止, 分母は Y0 そのまま)
  → FABSS = curve(ABS) * k                                    (相関式: 本付録の統一形)
  → 物理量 = SP_A*FABSS^2 + SP_B*FABSS + SP_C                 (スパン校正)
```

- 機差補正 kiza は廃止済みであり、演算チェーンは「相関式ベース → スパン校正」の 2 段である。
- 透視度のみベースが累乗式で、その上の 2 次補正が SP 相当である (構造は共通、ベース形のみ異なる)。


## 付録D 調整機能とパラメータ記憶

**付録D 調整機能とパラメータ記憶**

本付録は、IM-110 の基板調整・校正で作られる全パラメータの生成手順と、その記憶方式（プローブフラッシュ統合ストア一本化＝T1）を規定する。記述は現行実装（本体 `IM_110.c` / プローブ `IM_110.c` / `probe_store.h`）と真実源（`mlss-calc-reference.md` §12・§A-2・§8、`protocol-rs232c.md` §3.2.6）に一致させている。

---

**D.1 責務分界（プローブと本体）**

- プローブは AD 値（mV）をそのまま出力し、係数・調整パラメータを自身のフラッシュに保管するだけである。補正・濃度換算の計算は行わない。
- 本体は、プローブから受け取った AD 値に対して吸光度化・ベース式・温度補正・スパン校正を適用し、物理量（mg/L, cm 等）を算出する。
- プローブ側の計算で使うパラメータは、空中出力を 1700mV に正規化する span 傾き `ADC_Span[]` と LED PWM duty `LED_Out[0]` のみである。それ以外の校正係数はプローブが「保管するだけ」で、適用計算はすべて本体で行う。

---

**D.2 調整順序（出荷時／基板調整）**

調整は次の順序で実施する。各段の出力が次段の前提となるため、順序を入れ替えてはならない。

| 順 | 調整 | 対象 ch | 内容 | 保存先 |
|---|---|---|---|---|
| 1 | LED PWM 調整 | ch1〜4 | 空中で ch1〜4 の生 mV の最大値が 1700mV になるよう LED duty を収束させる（2048mV 振り切れ回避）。ch5（透視度）は対象外 | 統合ストア Page1 `LED_Out` |
| 2 | MLSS AD0（SADS） | ch1〜5 | 空中の生 mV を各 ch の span 基準 `ADC_Span[ch]` に記録する。以後 出力 = 生mV × 1700 / ADC_Span で空中出力が 1700mV へ正規化される。取得ゲートは生 mV 1400〜2048mV、範囲外は NG | 統合ストア Page1 `ADC_Span[5]` |
| 3 | Ref 温度補正 | — | 5 / 20 / 35℃ の清水中 Ref 出力から 2 次式（相対 mV・原点通過）を作り、MLSS・SS を常に 20℃換算へ補正する。TR は SS の温度補正を流用する | 統合ストア Page3（MLSS）／Page10（SS） |
| 4 | 出荷時 3 点調整 | — | ABS→mg/L の 2 次式（No.1 基準カーブ）を 3 点フィットで確定する。温度補正込みの ABS を使うため、必ず段 3 の後に行う | 統合ストア Page4（MLSS）／Page11（SS）／Page13（TR） |

- 機差補正（kiza）は廃止した。個体差は上記 LED PWM・ADSpan・出荷時 3 点調整の 3 段で吸収するため、別途の 1 点比例補正は持たない。本体の測定チェーンは「ベース式 → スパン校正」の 2 段で、`×kiza` は存在しない。

---

**D.3 記憶方式（統合ストア一本化＝T1）**

- 真実源はプローブフラッシュである。全係数（現場での校正上書き分を含む）をプローブが保持する。本体故障交換時も再調整は不要、プローブ交換時のみ再調整となる。
- 記憶の実体は統合ストア `probe_store_t`（512B ＝ 16 ページ × 32B）ただ 1 つである。プローブフラッシュ上では `STORE_AD`（Page63, アドレス 0x0801F800, 2KB ページ）に置く。旧 2 領域構成（旧 PARAM ＝ Page62）および旧係数 vault は撤去済みで、参照・書込ともに行わない。
- 本体 EEPROM（M95256）にはこの 512B を**バイト同一配置**でミラーする。ミラーは EEPROM の page61〜76（16 ページ）を占有し、相対レコード N が プローブ Page N とバイト単位で完全一致する（両 MCU とも little-endian ARM＋IEEE754 単精度のためバイト表現が一致）。ミラーは非権威キャッシュである。
- 各 32B ページの末尾 1 バイト（offset 31）は当該ページ [0..30] の XOR チェックサムである。ストア全体は Page0 ヘッダの checksum（4B, 各ページ XOR バイトと自身の 4B を除いたペイロード総和）で担保する。読出時は magic／ver 一致・全ページ XOR 一致・checksum 一致の全成立で有効と判定する。

---

**D.4 転送プロトコル（32B ページ単位）**

プローブ⇔本体の係数・パラメータ転送は 32B ページ単位の 3 コマンドに一本化した。旧 RCF／WCF／WCS／WCFC 系（および ver2 の WCM／WCK／WCZ／WCT）はすべて廃止し、プローブは受理しない。ワイヤ表現は 1 ページ＝64 hex 文字（32B を昇順に %02X）で、フィールド単位の float テキスト化はしない（意味は共有 struct が担保する）。

| コマンド | 方向 | 機能 |
|---|---|---|
| RPG | 本体←プローブ | 統合ストアの読出。`RPG` で全 16 ページ一括（PGV 1 行＋PG 16 行＋OK）、`RPG,<N>` で単一ページ。ストア無効時は NG のみ |
| WPG | 本体→プローブ | `WPG,<N>,<64hex>` で RAM 像の 1 ページのみ更新。ページ末尾 XOR を検証し、不一致は NG |
| WSC | 本体→プローブ | commit。RAM 像全体を封緘（magic/ver/checksum/全ページ XOR）して STORE_AD へ 512B 一括（全ページ消去→書込）確定 |

本体側の読出フロー（`Probe_ReadCoef`）:
1. MS,0 で測定ストリームを止め、`RPG` を発行して全 16 ページを本体側ストア像 `g_hstore` へ組み立てる。
2. `store_valid` で検証し、有効なら `store_unpack_to_globals` で計算グローバル（MLSS_ZR / MLSS_Coef_ADZR / MLSS_Coef_TempC / SP_A/B/C / Mode_CF / SetVal 等、SS・TR も同様）へ写像する。
3. `mirror_save` で本体 EEPROM ミラー（page61-76）へバイト同一保存する。
4. ストア無効時（新品・magic 不一致・checksum/XOR NG）は戻り値 1 を返し、本体が自ミラー（または既定）を WPG＋WSC で書き込む＝マイグレーション。

本体側の書戻しフロー（`Probe_SyncStore`）:
1. ミラー（または直近 RPG 像）を土台に `store_pack_from_globals` で live 係数を詰め、`store_seal` で封緘する。
2. MS,0 の後、16 ページを `WPG,<N>,<64hex>` で順次送り、最後に `WSC` で確定する。
3. `mirror_save` で本体ミラーも同一内容へ更新する。

---

**D.5 フラッシュ レコード割付（統合ストア 16 ページ）**

`probe_store_t` は次の 16 ページで構成される。特記なき値は float（4B）。各ページ末尾 offset31 は XOR。base/cal ページは 7 float 制約に収めるため SetVal を持たず、校正点濃度は Page15 に集約した。

| Page | 機能 | 内容（32B, 末尾1B=XOR） |
|---|---|---|
| 0 | ヘッダ | magic(4B)＋ver(1)＋rsv(1)＋Probe_ID(4B)＋最終更新日(4B, packed YYYYMMDD, 0=未設定)＋Product_Name(7B)＋rsv(6)＋checksum(4B) |
| 1 | プローブ共通調整 | ADC_Span[ch1-5]（空中1700 span傾き）, LED_Out, k_depth |
| 2 | プローブ設定 | SADA, SADC, 選択中相関式No.(MLSS), Meas_Mode（int/設定、割付は要詰め） |
| 3 | MLSS ゼロ/温度 | ZR_出荷時, ZR_現場(AZC上書き), refZR, B, B2 |
| 4 | MLSS ベース/校正 | 出荷時2次(c0,c1,c2), SP_A, SP_B, SP_C |
| 5–9 | MLSS No.21-30 | 独立2次式×10（(c0,c1,c2)を2式/ページ）＝5ページ。初期値=Page4ベースのコピー |
| 10 | SS ゼロ/温度 | ZR_出荷時, ZR_現場, refZR, B, B2 |
| 11 | SS ベース/校正 | 出荷時2次(c0,c1,c2), SP_A, SP_B, SP_C |
| 12 | SS No.21 | 独立2次式×1 (c0,c1,c2) ＋予備 |
| 13 | TR ゼロ/ベース/校正 | TRZR, 累乗a,b, SP_A, SP_B, SP_C（温度補正=SS参照） |
| 14 | TR No.21 | 校正後 累乗a,b ＋予備 |
| 15 | 校正点濃度 SetVal | MLSS/SS/TR × (SetVal1, SetVal2)＝6 float |

- k_No（No.1-20 のゲイン progression）は全プローブ共通の FW 定数表であり、フラッシュには保存しない（保存するのは Page2 の選択中 No. のみ）。
- 相関式初期化・ゼロ初期化は、Page4/Page11 の出荷時ベースおよび Page3/Page10 の ZR_出荷時を、対応する現場枠・No.21-30 スロットへコピーする本体側処理である。

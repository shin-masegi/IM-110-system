### C/D. A/DC ゼロ/スパンの扱い — 決定 2026-07-13
現状プローブは `送信値 = (生mV − ADC_Zero) × 1850 / ADC_Span`（`IM_110.c:205`）で正規化。
`ADC_Zero`=暗時(SADZ)、`ADC_Span`=基準点(空気中1750, SADS)。

- **スパン正規化（空気中1750）は削除**（冗長）。
  - 理由: 濃度は吸光度 `ABSS=log10(ZR/測定mV)`＝**比**。器差/濃度校正で ZR(清水0mg/L)と8000mg/L の
    2点をそのプローブで取れば対応が確定し、前段のスパン正規化は比でキャンセルする。空気より清水基準が素直。
  - 「清水≒1750mV」目標は **LED duty（明るさ）調整**側の役目として残す（ADC動作点/飽和/SN比）。
- **ゼロ（暗時/遮光 ADZR）は残す**が、**本体側で一本化する（後で実施）**。
  - 現状: 暗時ゼロ引きが プローブ `ADC_Zero` と 本体 `MLSS_ADZR` の**2箇所**にある → 本体側の単一処理に寄せる。
  - **ゼロは相関式に依存しない「同一係数」に統一**する（暗時オフセットはプローブ固有＝相関式の選択と無関係）。
    相関式 No. を切り替えてもゼロは共通の1値。
  - **MLSS / SS / 透視度 すべて同じ考え方**（各測定項目ごとにゼロは単一係数、相関式非依存）。
  - タイミング: **後で一本化**（他の論点を先に進めてよい）。
- 実装メモ（後日）: プローブ `ADC_Zero` の暗時引きと本体 `MLSS_ADZR` の二重を解消。
  vault 保存も、ゼロは per-eq ではなく **mode 単位の単一値**として持つ（現行 vault は ZR を per-eq 保存＝別物。
  ゼロ(ADZR)は setval/共通域に単一で置く方向。将来のレイアウト検討）。

### E. Ref 傾き係数（温度補正） — 決定 2026-07-13
手順4。**Ref を温度プロキシにして受光 mV を補正**する（運転時に温度計不要。Ref は測定値6chの1つとして本体に届く）。

- 取得: 5/20/35℃ 清水で Ref・受光を取得。`RefADZR`（Ref暗時ゼロ）を引いた値から補正式を作る。
- 実データ（`Ref傾き係数作成`）: 5℃(Ref1604,受光1968)/20℃(1477.6,1792)/35℃(1346.7,1616.8)。
  受光vsRef の傾き = 5→20℃ **1.393**、20→35℃ **1.339**（約4%変化）→ 1次式1本では誤差。
- **フィット方式 = 案②「2次式3点通過」を採用**（既存 `MLSS_Calib_Span` の3点ソルバ流用＋滑らかさ）。
  - **実機で中間温度・範囲外(0/40℃付近)に誤差が出たら、案①「2本の1次式（20℃ピボット）」形式へ切替**える。
  - 範囲外は当面クランプ想定（実機評価で判断）。
- **適用式 = 加算補正（20℃相当へ引き戻す）**（確定）: `受光_corr = 受光 + a·(Ref_20 − Ref)`。
  - `Ref_20`=20℃時の基準 Ref、`a`=Ref傾き係数（案②なら Ref を変数にした2次補正）。
- **透視度の温度補正 = SS の補正式をそのまま流用**（確定）。
  - 根拠: 透視度は SS と同じ光学ch（`SADC,1100`）を **SEL3 ON=PGA x2 ゲイン**（`SS,1`）で読むだけ、**Ref も共通**。
  - よって温度補正係数セットは **MLSS と SS の2セット**。透視度は SS のを参照（透視度固有は濃度側の累乗式のみ）。
  - **加算補正のため x2 ゲインは効く** → **補正は SS の“ゲイン前”信号に掛けてから x2 する**のが正解（SS 式をそのまま流用可）。
    TR の post-gain 値に直接掛ける実装にする場合は傾き `a` を **2倍**する必要あり。実機で要確認。
- **確定事項（実装共通）**:
  - `RefADZR`（Ref暗時ゼロ）取得が必要 → **ゼロ一本化（C/D）に Ref 分も含める**。
  - 温度補正係数は**相関式非依存＝mode単位の単一係数**（ゼロと同性質）。→ **現行 vault に無い新フィールド追加**が必要
    （vault は ZR等 per-eq / setval per-mode。温度補正は per-mode で MLSS/SS の2セット追加）。
  - 測定パイプラインに**新ステップ**: `Calc_MLSS/Calc_SS` の吸光度計算の**前**に「Ref による受光補正」を挿入。
    現状 `FABS_SPAN`（単純係数）とは別段。
  - 係数算出は**3点厳密解＝機上で計算可**（手順4-2 の 4次式6点最小二乗とは別で軽い）。
  - 取得画面は削除予定の `WAT05C/WAT20C/WAT35C` の置き換えに相当。

### F. 1次演算式（相関式）の回帰 — 決定 2026-07-13：**機上で最小二乗**
現状 `MLSS/SS/TR_Mode_CF[30][7]` は **FW ハードコード初期化テーブル**（`IM_110.c:108/146/201`、RAM上で書換可だが
EEPROM/vault 未保存＝毎起動で初期値に戻る）。回帰結果はこの Mode_CF に入る。

- **解法**:
  - 2点1次式（黒mode1）: 厳密解。
  - 6点4次式（白mode5）: 正規方程式 **5×5 → ガウス消去**。
  - 6点累乗式（透視度）: **log 線形化** `ln y = ln a + b·ln x` → 線形LS（良条件）。
- **数値安定性（必須対策）**: 本機 FPU は**単精度のみ**（`-mfpu=fpv4-sp-d16`）。4次正規方程式は悪条件。
  - ① 吸光度 x を**中心化・スケーリング**してから当てはめ（係数は後で戻す）。
  - ② 解く部分は **double（ソフト浮動小数）**（校正時1回のみ、数ms）。
  - ③ **残差/R² を算出して妥当性チェック**（悪ければ弾く）。
- **機上フロー**: ADBOAD で 6 溶液ぶん (吸光度, 基準器mg/L) を安定キャプチャ → 機上で4次フィット → Mode_CF を vault へ。
- **→ 保存ギャップ（H で確定）**: Mode_CF は現状どこにも永続化されていない。**per-eq で vault に Mode_CF（枠7 float）を追加保存**
  する必要がある（＋EEPROM キャッシュ）。これで「No.21-30 を機上校正して式ごと書換」が成立（従来の校正は ZR/SP 調整のみで式形固定だった）。

### I. 調整中の生データ表示 — 決定 2026-07-13
- **機上（ADBOAD 画面）に受光/Ref の生 mV を表示**する。PC ログは対象外（機上のみ）。

### H. vault 保存レイアウト — 決定 2026-07-13
決定済み①〜④を現行 vault(ver1, 平坦 `mlss[10][7]`) に追加するため **構造体化＋ver2** にする。

```c
// 相関式(No.)ごとの係数
typedef struct {
  float cf[7];      // 既存: ZR,FABS_SPAN, SP_A/SP_B/SP_C(=現場スパン校正), 1A,1B
  float modecf[7];  // ① 相関式 Mode_CF (最大6次=7係数)             ← F
  float kiza;       // ④ 機差補正 倍率 (工場, 1点比例)              ← G
} eq_coef_t;        // 15 float = 60B

// 測定モードごとの係数 (0=MLSS,1=SS,2=TR)
typedef struct {
  float setval[2];  // 既存: 校正点濃度 SetVal_1/2
  float adzr[2];    // ③ ゼロ: 受光ADZR, RefADZR (per-mode 単一)    ← C/D
  float tempc[7];   // ② 温度補正 (7係数=1 EEPROMページ相当・最大枠) ← E
} mode_coef_t;      // 11 float = 44B

typedef struct {
  uint32_t magic; uint16_t ver /*=2*/; uint16_t rsv; uint32_t rsv2/*8境界padding*/;
  eq_coef_t   mlss[10];   // No.21-30
  eq_coef_t   ss[10];
  eq_coef_t   tr;         // 相関式無し1式
  mode_coef_t mode[3];    // MLSS/SS/TR
  uint32_t checksum;
} probe_vault_t;          // 合計 ~1408B (2KB Page63 内, 8の倍数)
```

- **温度補正 tempc[7]**: 1 EEPROMページ(32B=7float)に収まる最大枠で確保（実際は案②2次式なので係数はその範囲で使用、余りは0）。
- **TR の tempc / adzr は SS 参照（実体持たず）**: `mode[2]` のこの2つは未使用。実行時に SS(`mode[1]`)のを流用（TRは同一光学ch+PGA x2、Ref共通）。`mode[2].setval` は使用。
- **protocol 拡張（命名確定）**: RCF/WCF/WCS/WCFC(§3.2.6) を ver2 化。行種を追加 —
  読み: `CM`=Mode_CF, `CK`=機差補正, `CZ`=ADZR, `CT`=温度補正（既存 `CF`=cf, `CS`=setval に追加）。
  書き: `WCM` / `WCK` / `WCZ` / `WCT`（既存 `WCF`/`WCS`/`WCFC`）。CM/CK/WCM/WCK は per-eq(mode,eq)、CZ/CT/WCZ/WCT は per-mode(mode)。
- **移行**: ver1→ver2 は magic/ver 不一致で本体が検出し、既存 ver1 の cf/setval を引き継ぎ、新フィールドは初期値(0/未校正)。

**→ H 決定完了。残りは I（調整中の生データ表示）のみ。**

### G. 機差補正 / スパン校正 — 決定 2026-07-13：**別係数で分離（案Y）**
「工場で基準器に合わせる機差補正」と「現場のユーザースパン校正」は**別物**として扱い、**記録値も別係数**で保存する。

- **機差補正**: 工場・**ADBOAD** で実施。手順書どおり**1点比例**（`倍率 = 基準器mg/L ÷ 1次演算後mg/L`）。基準器(IM-100P)に合わせる。
- **スパン校正**: 現場・**既存の2点/3点後校正**（`SP_A/B/C`, ADCAL/MCAL）。**そのまま維持**。
- **パイプライン**: `吸光度 →[Mode_CF]→ FABSS →[機差補正(工場)]→[スパン校正(現場)]→ 最終mg/L`。
  機差補正を先に掛けて基準器基準にし、その上に現場スパン校正が乗る**多段**。現場校正しても工場機差補正は消えない。
- **保存**: 機差補正係数を **vault に新規追加**（既存 `SP_A/B/C`＝現場スパン校正 とは別フィールド）。
- 要確認（実装時）: 機差補正係数は **per-eq**（手順は相関式ごと＝per-eq 寄り）か per-mode か。式は**1点比例（倍率のみ）**で確定か。

## 現行モデル（これが正）

修理で本体基板のみ交換・プローブ付替をしても測定値が正しく出るよう、**プローブ性能に関わる係数をプローブ flash に持たせ真実源とする**（設計合意済み）。

- **保存レイアウト**: `probe_store_t`（512B = 16 ページ × 32B, Page63/`STORE_AD`）。1 機能 = 32B ページ。ページ意味は §12（Page0=ヘッダ/ID/最終更新日, Page1=ADC_Span/LED_Out/k_depth, Page2=設定, Page3-15=MLSS/SS/TR のゼロ・温度・ベース・校正・No.21-30・SetVal）。
- **本体 EEPROM ミラー**: プローブ領域と**バイト完全同一配置**（相対レコード N == プローブ page N, §A-2）。ページオフセットも同一。非権威キャッシュ。
- **転送**: 接続時に本体が `RPG` で全 16 ページ読込→RAM 反映→EEPROM ミラーへ同期。校正確定時は変更ページを `WPG`＋`WSC` で書戻し。プローブ無応答時は本体ミラーへフォールバック（§A-2 3 層 + ID/最終更新日 判定）。
- **FW 固定**: 相関式 No.1-20（校正不可ハードコード、保存対象外）。

## 旧モデル（退役・参考）

初期は係数を PARAM とは別の専用領域（Page63）に独立構造で保存し、per-eq/per-mode の意味論コマンドで転送する設計だった（`cf[7]`=ZR/FABS_SPAN/SP_A/SP_B/SP_C/1A/1B、`modecf`、機差補正 `kiza`、per-mode の SetVal/ADZR/温度）。これは:

- **旧測定モデル**の係数枠であり、現行の再設計モデル（1700 span・2 次式×k_No・TR 累乗・**`kiza` 廃止**, §8）と非互換。
- 別領域が統合ストアと同じ Page63 を奪い合い衝突していた（`handoff.md` §4 参照）。

→ **§12 統合ストアの係数ページ（Page3-15）へ吸収し、旧意味論コマンドは §3.2.6 のページ転送へ一本化**する方針に置換済み。旧設計の詳細はコミット履歴を参照。

## How to apply

実装は CLAUDE.md §3.1/§3.2 に従う。**まず `protocol-rs232c.md §3.2.6`（`RPG`/`WPG`/`WSC`）を真実源として更新済み** → プローブ FW（統合ストアの係数ページ読み書き・旧係数コード削除）→ 本体 FW（`RPG` 受信・EEPROM ミラー同一配置化・新モデル計算）を同一セッションで実装。残作業は `handoff.md` §6（T1）。ハード根拠は `hardware-truth-source.md`、通信経路は `probe-fup-bridge.md` と同じ USART2↔本体。

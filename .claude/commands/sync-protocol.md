# /sync-protocol

`docs/protocol-rs232c.md` の定義と、本体 (`../IM-110/`) ・プローブ (`../IM-110_Probe/`)
両実装との食い違いを検出します。

## 前提

- セッション内で `/add-dir ../IM-110` と `/add-dir ../IM-110_Probe` が済んでいること
- 済んでいなければ実行前にユーザーに促すこと

## 手順

1. `docs/protocol-rs232c.md` を読み、以下を抽出する:
   - コマンド一覧 (CMD hex → 名前)
   - 各コマンドの要求パラメータ / 応答データ構造
   - タイムアウト値
   - エラーコード一覧

2. 本体側のコードで以下を探す:
   - コマンド ID の `#define` または `enum` 定義 (`grep -rn "0x[0-9A-F]\{2\}" ../IM-110/include/ ../IM-110/src/`)
   - コマンドパーサ / ディスパッチャ (`switch` / ハンドラテーブル)
   - タイムアウト定数 (`TIMEOUT`, `MS` などのキーワード)

3. プローブ側のコードで同じものを探す

4. 以下の観点でクロスチェック:
   | チェック項目 | 不一致時の扱い |
   |---|---|
   | 本体定義のCMDが仕様にあるか | 警告 |
   | 仕様のCMDが本体に実装されているか | 情報 (未実装かも) |
   | 仕様のCMDがプローブに実装されているか | 情報 (未実装かも) |
   | 本体のCMD定義とプローブのCMD定義が一致するか | **エラー** |
   | タイムアウト値が仕様と実装で一致するか | 警告 |

5. レポートを markdown で出力する。フォーマット例:

```markdown
# Protocol Sync Report (YYYY-MM-DD)

## Summary
- 仕様コマンド数: 12
- 本体実装: 10 / 12 ✓
- プローブ実装: 11 / 12 ⚠
- 不整合 (Error): 0
- 警告 (Warning): 2

## Errors
(なし)

## Warnings
- [W1] GET_TEMP のタイムアウトが仕様 500ms、本体実装 300ms
  - 仕様: docs/protocol-rs232c.md §4
  - 実装: ../IM-110/src/comm.c:142

## Missing implementations
- [本体] 0x31 GET_TEMP: 未実装
- [プローブ] 0x7F ERROR (自律送信): 未実装
```

## 重要な注意

- 実装を勝手に書き換えないこと。レポート出力のみ。
- ユーザーから明示的に「修正して」と言われた場合のみ、差分提案モードに入る。
- サブリポジトリが add-dir されていない場合は、該当側のチェックをスキップし
  レポートにその旨を明記する。

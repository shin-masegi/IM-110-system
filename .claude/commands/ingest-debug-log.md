# /ingest-debug-log

`docs/debug-log/` 配下の未処理エントリ（`_processed/` 以外の `.md` ファイル）を
読み込み、内容を恒久的な文書に反映する作業を支援します。

## 手順

1. `docs/debug-log/` 直下の `.md` ファイルを一覧する
   （`TEMPLATE.md` と `README.md` は除外、`_processed/` 配下は除外）
2. 各ファイルの front matter の `status` が `new` のものを処理対象にする
3. 各エントリについて:
   a. `事実` と `検証した結果` から、**確定した知見** を抽出
   b. `仕様への反映候補` セクションがあればそれを叩き台にする
   c. 反映先候補を以下から選ぶ（複数可）:
      - `docs/protocol-rs232c.md` ← プロトコル仕様に関わる場合
      - `IM-110-system/CLAUDE.md` ← システム横断のルール・知見
      - `../IM-110/CLAUDE.md` ← 本体固有の知見（`/add-dir ../IM-110` が必要）
      - `../IM-110_Probe/CLAUDE.md` ← プローブ固有の知見
      - 各リポジトリ内のソースコメント
   d. 差分案を提示し、ユーザーの承認を得る
   e. 承認後、実際に編集を行う
4. 取り込み完了したエントリは:
   - front matter の `status` を `processed` に更新
   - `docs/debug-log/_processed/` に移動
5. 最後に変更サマリを提示し、コミットメッセージ案を出す
   （各リポジトリが別なので、コミットは **リポジトリ単位に分ける** こと）

## 重要な注意

- **仮説** は恒久文書に書き込まないこと。確定した知見だけを反映する。
- プロトコルに影響する変更を提案する場合、必ず本体・プローブ両側への影響を確認する。
- ユーザーの承認なしに `.md` ファイル以外（ソースコード等）を編集しない。
- サブリポジトリ (`../IM-110`, `../IM-110_Probe`) が `/add-dir` されていない場合、
  該当リポへの反映は「後で手動で行ってください」とリストを出すに留める。

## 出力形式

処理開始時に以下を表示してください:

```
デバッグログ取り込み開始
未処理エントリ: N 件
  - 2026-04-15-mcp3424-drift.md
  - 2026-04-18-ms-autosend-jitter.md
  ...
```

各エントリ処理時:

```
[1/N] 2026-04-15-mcp3424-drift.md
  確定した知見:
    - (箇条書き)
  反映先案:
    - docs/protocol-rs232c.md §4 タイミング: MS 自律送信周期に ±50ms のジッタ実測 → 注記追加
    - ../IM-110_Probe/CLAUDE.md §6: ADC 移動平均の推奨値を実測値で更新
  反映しますか? (y / n / e=編集)
```

## サンプルエントリの扱い

`docs/debug-log/2026-04-22-sample-entry.md` は動作確認用のサンプルで、本文に「反映不要、
`_processed/` に移動するだけでよい」と明記されている。このコマンドで未処理一覧として
検出されたら、**反映先案として「サンプル — 移動のみ」と表示し、ユーザー承認後に `_processed/`
に移動** する（front matter の `status` 更新は不要、サンプルなのでそのまま移動）。

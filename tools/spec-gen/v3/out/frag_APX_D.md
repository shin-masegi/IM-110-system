**付録D 調整機能とパラメータ記憶**

本付録は、IM-110 の基板調整・校正で作られる全パラメータの生成手順と、その記憶方式（プローブフラッシュ統合ストア一本化＝T1）を規定する。記述は現行実装（本体 `IM_110.c` / プローブ `IM_110.c` / `probe_store.h`）と真実源（`mlss-calc-reference.md` §12・§A-2・§8、`protocol-rs232c.md` §3.2.6）に一致させている。

---

**D.1 責務分界（プローブと本体）**

- プローブは AD 値（mV）をそのまま出力し、係数・調整パラメータを自身のフラッシュに保管するだけである。補正・濃度換算の計算は行わない。
- 本体は、プローブから受け取った AD 値に対して吸光度化・ベース式・温度補正・スパン校正を適用し、物理量（mg/L, cm 等）を算出する。
- プローブ側の計算で使うパラメータは、空中出力を 1700mV に正規化する span 傾き `ADC_Span[]` と LED PWM duty `LED_Out[0]` のみである。それ以外の校正係数はプローブが「保管するだけ」で、適用計算はすべて本体で行う。

---

**D.2 調整順序（出荷時／基板調整）**

調整は次の順序で実施する。各段の出力が次段の前提となるため、順序を入れ替えてはならない。

| 順 | 調整 | 対象 ch | 内容 | 保存先 |
|---|---|---|---|---|
| 1 | LED PWM 調整 | ch1〜4 | 空中で ch1〜4 の生 mV の最大値が 1700mV になるよう LED duty を収束させる（2048mV 振り切れ回避）。ch5（透視度）は対象外 | 統合ストア Page1 `LED_Out` |
| 2 | MLSS AD0（SADS） | ch1〜5 | 空中の生 mV を各 ch の span 基準 `ADC_Span[ch]` に記録する。以後 出力 = 生mV × 1700 / ADC_Span で空中出力が 1700mV へ正規化される。取得ゲートは生 mV 1400〜2048mV、範囲外は NG | 統合ストア Page1 `ADC_Span[5]` |
| 3 | Ref 温度補正 | — | 5 / 20 / 35℃ の清水中 Ref 出力から 2 次式（相対 mV・原点通過）を作り、MLSS・SS を常に 20℃換算へ補正する。TR は SS の温度補正を流用する | 統合ストア Page3（MLSS）／Page10（SS） |
| 4 | 出荷時 3 点調整 | — | ABS→mg/L の 2 次式（No.1 基準カーブ）を 3 点フィットで確定する。温度補正込みの ABS を使うため、必ず段 3 の後に行う | 統合ストア Page4（MLSS）／Page11（SS）／Page13（TR） |

- 機差補正（kiza）は廃止した。個体差は上記 LED PWM・ADSpan・出荷時 3 点調整の 3 段で吸収するため、別途の 1 点比例補正は持たない。本体の測定チェーンは「ベース式 → スパン校正」の 2 段で、`×kiza` は存在しない。

---

**D.3 記憶方式（統合ストア一本化＝T1）**

- 真実源はプローブフラッシュである。全係数（現場での校正上書き分を含む）をプローブが保持する。本体故障交換時も再調整は不要、プローブ交換時のみ再調整となる。
- 記憶の実体は統合ストア `probe_store_t`（512B ＝ 16 ページ × 32B）ただ 1 つである。プローブフラッシュ上では `STORE_AD`（Page63, アドレス 0x0801F800, 2KB ページ）に置く。旧 2 領域構成（旧 PARAM ＝ Page62）および旧係数 vault は撤去済みで、参照・書込ともに行わない。
- 本体 EEPROM（M95256）にはこの 512B を**バイト同一配置**でミラーする。ミラーは EEPROM の page61〜76（16 ページ）を占有し、相対レコード N が プローブ Page N とバイト単位で完全一致する（両 MCU とも little-endian ARM＋IEEE754 単精度のためバイト表現が一致）。ミラーは非権威キャッシュである。
- 各 32B ページの末尾 1 バイト（offset 31）は当該ページ [0..30] の XOR チェックサムである。ストア全体は Page0 ヘッダの checksum（4B, 各ページ XOR バイトと自身の 4B を除いたペイロード総和）で担保する。読出時は magic／ver 一致・全ページ XOR 一致・checksum 一致の全成立で有効と判定する。

---

**D.4 転送プロトコル（32B ページ単位）**

プローブ⇔本体の係数・パラメータ転送は 32B ページ単位の 3 コマンドに一本化した。旧 RCF／WCF／WCS／WCFC 系（および ver2 の WCM／WCK／WCZ／WCT）はすべて廃止し、プローブは受理しない。ワイヤ表現は 1 ページ＝64 hex 文字（32B を昇順に %02X）で、フィールド単位の float テキスト化はしない（意味は共有 struct が担保する）。

| コマンド | 方向 | 機能 |
|---|---|---|
| RPG | 本体←プローブ | 統合ストアの読出。`RPG` で全 16 ページ一括（PGV 1 行＋PG 16 行＋OK）、`RPG,<N>` で単一ページ。ストア無効時は NG のみ |
| WPG | 本体→プローブ | `WPG,<N>,<64hex>` で RAM 像の 1 ページのみ更新。ページ末尾 XOR を検証し、不一致は NG |
| WSC | 本体→プローブ | commit。RAM 像全体を封緘（magic/ver/checksum/全ページ XOR）して STORE_AD へ 512B 一括（全ページ消去→書込）確定 |

本体側の読出フロー（`Probe_ReadCoef`）:
1. MS,0 で測定ストリームを止め、`RPG` を発行して全 16 ページを本体側ストア像 `g_hstore` へ組み立てる。
2. `store_valid` で検証し、有効なら `store_unpack_to_globals` で計算グローバル（MLSS_ZR / MLSS_Coef_ADZR / MLSS_Coef_TempC / SP_A/B/C / Mode_CF / SetVal 等、SS・TR も同様）へ写像する。
3. `mirror_save` で本体 EEPROM ミラー（page61-76）へバイト同一保存する。
4. ストア無効時（新品・magic 不一致・checksum/XOR NG）は戻り値 1 を返し、本体が自ミラー（または既定）を WPG＋WSC で書き込む＝マイグレーション。

本体側の書戻しフロー（`Probe_SyncStore`）:
1. ミラー（または直近 RPG 像）を土台に `store_pack_from_globals` で live 係数を詰め、`store_seal` で封緘する。
2. MS,0 の後、16 ページを `WPG,<N>,<64hex>` で順次送り、最後に `WSC` で確定する。
3. `mirror_save` で本体ミラーも同一内容へ更新する。

**D.5 フラッシュ レコード割付（統合ストア 16 ページ）**

`probe_store_t` は次の 16 ページで構成される。特記なき値は float（4B）。各ページ末尾 offset31 は XOR。base/cal ページは 7 float 制約に収めるため SetVal を持たず、校正点濃度は Page15 に集約した。

| Page | 機能 | 内容（32B, 末尾1B=XOR） |
|---|---|---|
| 0 | ヘッダ | magic(4B)＋ver(1)＋rsv(1)＋Probe_ID(4B)＋最終更新日(4B, packed YYYYMMDD, 0=未設定)＋Product_Name(7B)＋rsv(6)＋checksum(4B) |
| 1 | プローブ共通調整 | ADC_Span[ch1-5]（空中1700 span傾き）, LED_Out, k_depth |
| 2 | プローブ設定 | SADA, SADC, 選択中相関式No.(MLSS), Meas_Mode（int/設定、割付は要詰め） |
| 3 | MLSS ゼロ/温度 | ZR_出荷時, ZR_現場(AZC上書き), refZR, B, B2 |
| 4 | MLSS ベース/校正 | 出荷時2次(c0,c1,c2), SP_A, SP_B, SP_C |
| 5–9 | MLSS No.21-30 | 独立2次式×10（(c0,c1,c2)を2式/ページ）＝5ページ。初期値=Page4ベースのコピー |
| 10 | SS ゼロ/温度 | ZR_出荷時, ZR_現場, refZR, B, B2 |
| 11 | SS ベース/校正 | 出荷時2次(c0,c1,c2), SP_A, SP_B, SP_C |
| 12 | SS No.21 | 独立2次式×1 (c0,c1,c2) ＋予備 |
| 13 | TR ゼロ/ベース/校正 | TRZR, 累乗a,b, SP_A, SP_B, SP_C（温度補正=SS参照） |
| 14 | TR No.21 | 校正後 累乗a,b ＋予備 |
| 15 | 校正点濃度 SetVal | MLSS/SS/TR × (SetVal1, SetVal2)＝6 float |

- k_No（No.1-20 のゲイン progression）は全プローブ共通の FW 定数表であり、フラッシュには保存しない（保存するのは Page2 の選択中 No. のみ）。
- 相関式初期化・ゼロ初期化は、Page4/Page11 の出荷時ベースおよび Page3/Page10 の ZR_出荷時を、対応する現場枠・No.21-30 スロットへコピーする本体側処理である。

**D.7 補足（当面の出力）**

- 出荷時 2 次式データが未取得のため、No.1-20 のベースは当面 Mode_CF[0]（現行=線形係数 21315）で従来同等の出力となる。出荷時 3 点調整（AMC/AMCF）を実施すると本来の 2 次カーブに置き換わる。
- 実運用の MLSSZR（ゼロ基準）は、清水ゼロ校正（AZC）が空中正規化後の Y0 で上書きする。電源投入直後の既定値 1700 は校正前のプレースホルダである。

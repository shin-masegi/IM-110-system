# 相関式・項目別演算 事実ソース（付録用）

> 真実源: `docs/protocol-rs232c.md` §3.2.6（vault）、`IM-110/Core/Src/Normal.c`（相関式選択）、
> `IM補正式作成手順.xlsx`。以下の事実のみを使い、係数値や式を創作しない。

## 測定値の演算チェーン（全項目共通の流れ）
生 mV（プローブ MCP3424）→ 吸光度(ABS) → 1次演算（相関式係数で物理量へ）→ スパン校正 → 表示値。

- **吸光度 (ABS)** は項目別の ZR を用いる:
  - `ABS = Log( (項目ZR + (現在のref出力 - refZR) * ref補正傾きB) / (Y0 - ADZR) )`
  - MLSS の場合 項目ZR = `MLSSZR`、透視度の場合 項目ZR = `透視度ZR`。
- **1次演算**: 吸光度 ABS から相関式係数で物理量を算出（MLSS/SS は多項式=4次式相当、透視度は累乗/1次式）。係数は「相関式」で選ぶ。
- **スパン校正係数 SP_B** = `スパン校正値(mg/L) / 1次演算後(mg/L)`。1次演算結果に SP_B を掛けて校正後の値を得る。

## 相関式（Correlation equation）とは
- 吸光度から物理量へ換算する式の**プリセット**。設定メニューの「相関式」で **No. を選択**（`corr_sel`, Normal.c）。No. は 01〜30 の2桁入力。
- **No.01〜20（内部値 0〜19）= 係数 FW ハードコード固定・校正不可**。
- **No.21〜30（内部値 21〜30）= 係数をプローブ vault に保存・校正可能**。
- 相関式を切り替えると、該当係数をプローブ vault から再読込し EEPROM キャッシュへ同期する。
- 「校正不可な相関式（内部値 < 20）」では校正メニューの 2点/3点 を非表示にし、ゼロとリセットのみ巡回。

## 項目（mode）別の相関式構成（protocol §3.2.6 vault）
- `<mode>`: 0=MLSS / 1=SS / 2=TR(透視度)。
- 各相関式の係数 = **7 float**: `ZR, FABS_SPAN, SP_A, SP_B, SP_C, 1A, 1B`。
- **MLSS**: 相関式 No.21〜30 の 10 式（10 × 7 float）。
- **SS**: 相関式 No.21〜30 の 10 式（10 × 7 float）。
- **TR(透視度)**: 相関式無し、1 セットのみ（eq=0 固定、1 × 7 float）。
- **界面(DEPTH)**: **相関式に依存しない**（corr_idx=0 扱い、2点/3点校正は常に非表示）。
- 校正点濃度: mode 別に `Cal_SetVal_1 / Cal_SetVal_2`（MLSS の 3点中間は Cal_SetVal_2）。

## 係数の真実源と転送
- 係数の真実源は**プローブ flash（vault）**。本体 EEPROM はキャッシュ。本体は接続時に **RCF** で読み出す（protocol §3.2.6）。
- 書込みは `WCF`（1式ぶんの係数）/ `WCS`（校正点濃度）→ `WCFC` で flash 確定。vault 無効時は EEPROM キャッシュへフォールバック。

## 水温補正との関係
- 水温補正（TLsp/TLzr/THsp/THzr/TOsp/TOzr, Calc.c）は**全モード共通**で、測定項目(MLSS/SS/透視度)に依存しない。上記の項目別演算とは別系統。

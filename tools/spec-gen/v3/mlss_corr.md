## 3. 1次演算: ABS → FABSS（④相関式 Mode_CF）

`IM_110.c:1007-1013`（6次までの多項式, ホーナーではなく明示展開）:

```
FABSS = Σ(k=0..6) MLSS_Mode_CF[MLSS_MODE][k] × ABS^k
```

- `MLSS_MODE` = 選択中の相関式テーブル index（0=No.1 … 19=No.20 … 29=No.30）
- **現状テーブルは全て「1次直線」**（`MLSS_Mode_CF[..][1]=傾き`, 他 0）⇒ 実質 `FABSS = C1 × ABS`
- **No.1-20 = FW 固定・校正不可**（等間隔グラデ, C1: No.1=21315 … No.11=42494.4572(=従来黒No.5) … No.20=61555.97）
- **No.21-30 = 校正可**（初期 C1=21315=No.1 傾き, ADBOAD 校正で vault modecf を上書き → `Apply_Vault_To_Live` で反映, `:956`）

### 第2領域: 出荷時 3 点校正係数（4次式）— プローブ調整の第3ステップ
プローブ調整（LED PWM → ADZero）の後に実施する。

- **動作は現行「MLSS 校正モード, 相関式 No.21〜30」と全く同じ** 3 点校正機構
  （[adj_fit_poly](../../IM-110/Core/Src/IM_110.c#L1244) 多項式フィット → `*_Vault_ModeCF` → `WCM` → `WCFC` 確定）。
- **この校正機能を ADBOAD にも追加**する（現状は CAL/校正モードのみ）。**MLSS / SS / TR すべて対象**。
- **出荷時補正係数 = MLSS / SS / TR それぞれ 1 式ずつ持つ**（＝出荷初期値係数）。
  - **実質 2 次式**（3 点で一意に決まるのは 2 次まで）。格納は `modecf[7]` 枠（最大6次）だが **3 次以上は 0**。
    ＝「係数枠が 4 次まであるだけ」で実体は 2 次。
  - 経緯: 元の MLSS白 6点→4次式（付録A）は実験式。**最終的に 1 次傾き補正（相関式 No.13 ≒ ゲイン2.19 相当）に集約**したため、
    出荷時は **3点→2次式で確定**（4次は使わない）。

### 相関式 No. モデル（改訂・決定: (B) 併存）

- **相関式係数は今回から「傾きのみ（1次スケール）」に簡素化**（元は独立した 3 点調整の 4 次式係数だった）。
- **出荷時 MLSS 3 点調整係数 ＝ 相関式 No.1 相当の 4 次係数**（＝基準カーブ）。
- **No.1〜20**: この **No.1（基準 4 次式）を基準に、傾きを順次大きくしていく**（No.1=基準, No.2→No.20 で傾き増）。
  - 各相関式が持つのは **傾き（スケール）係数のみ**。基準の 4 次カーブ形状は共通で、傾きだけが No. で変わる。
- **No.21〜30**: **独立した 2 次式**（ユーザ 3 点校正で作成、基準の代わりに使う）。**傾き(ゲイン) k は 1.00 固定**
  （傾きは 2 次式係数に内包されるため追加ゲイン不要）。← (B) 併存。

**統一形（No.1〜30 共通）**:
```
出力(mg/L) = curve(ABS) × k
  No.1〜20 : curve = 出荷時2次式(基準・共通),  k = k_No（No.1=1.00 … No.20=2.8879）
  No.21〜30: curve = 各スロット独立2次式,       k = 1.00（固定）
```
**No.21〜30 の初期状態・初期化（決定 2026-07-20）**:
- **未調整（初期）状態**: 全10スロットに **出荷時3点校正係数（基準2次式）をコピー**、傾き **k=1.00**。
  ⇒ 未調整の No.21 選択でも No.1（基準×1.00）と同結果。**modecf が「全0」になる状態は通常存在しない**
  （現行の「全0=未校正フォールバック」ロジックに依存しない）。
- **相関式初期化機能**: No.21〜30 を上記初期状態（現在の出荷時係数を再コピー、k=1.00）へ戻す。
  ＝ ADBOAD/UART の新機能（既存 校正リセット C_RESET との関係は別途整理）。
- No.21-30 の運用上のセーフ機構の詳細は**未定**（今回はここまで）。
- ⚠ 現行実装は No.1-20 が FW 固定の 1 次直線（絶対傾き）テーブル（[IM_110.c:109](../../IM-110/Core/Src/IM_110.c#L109)）。
  本定義で **No.1 = 出荷時4次式、No.2-20 = その傾きスケール**に変わる。

### 傾きの掛け方（決定）＋ ゲイン係数変換表

- **決定**: 相関式 No. は **基準 4 次カーブ全体をゲイン倍**する。
  ```
  出力(mg/L) = 出荷時4次式(ABS) × k_No        （No.1: k=1.00）
  ```
- **抜けの指摘（要修正）**: 現行 `MLSS_Mode_CF[No.1-20]` は**絶対傾き**（No.1=21315…）で、ゲインモデルの
  No.1=1.00 になっていない。**No.1〜20 を「No.1=1.00 基準のゲイン係数 `k_No`」へ変換**しないと
  出荷時 4 次式（=No.1）がベースとして噛み合わず、出荷時 3 点調整が成立しない。

- **コード変更（予定・未実施）**: ① `MLSS_Mode_CF`/`SS_Mode_CF` No.1-20 を上記 `k_No` へ置換、
  ② 出力計算を `出荷時4次式(ABS) × k_No` に変更（現行の `Σ Mode_CF[k]·ABS^k` 直接評価から）。

# IM-110 事実パック（プローブ）

> プローブは**画面を持たない**。本体からRS-232Cコマンドで制御されるアナログ計測モジュール。
> ここに書かれた事実だけを使うこと。コマンド番号・ピン・チャネルを憶測で足さない。

## プローブ役割・電源（真実源: CLAUDE.md §4）
- MCU: **STM32G070KBT6**（Cortex-M0+, 64MHz, Flash 128kB / SRAM 36kB）。本体(STM32L4)とは系列が異なる。
- 電源: 本体CN4から`VIN`(5V)供給 → Q1(逆接保護) → `+05P`。バッテリー非搭載、単独動作不可。
  - `+05P` → MAX8881(IC14) → +3.3V → +3.3VA(アナログ)。
  - `+05P` → LM27761(IC15) インバーティングチャージポンプ → ±5V(アナログ `+05A`/`-05A`)。
  - LM336(IC9)+2SC2712 → +2.5V基準、分圧して`1V_REF`生成。

## アナログ計測フロントエンド（真実源: CLAUDE.md §4、回路図）
- オペアンプ: **TSX562 × 7個**（IC1〜IC7、低オフセット・レール to レールCMOS）。
- アナログMUX: **TMUX4053 × 2個**（IC10, IC11）。`SEL`(IC20.13/PA6) / `SEL3`(IC20.14/PA7)でチャネル切替。
- 外部ADC: **MCP3424（IC19）**、4ch・18bit ΔΣ、I2C1接続(SDA=IC20.1/SCL=IC20.32)。**STM32内蔵ADCは計測に不使用**。
- 入力保護: 各差動入力に1N4148WSクランプ。
- 主要信号系ネット名: `MLSS_JUKO` / `MLSS_REF` / `TRA_JUKO` / `TRA_REF`。
- 気圧センサ HSPPAD143A(I2C2)は**現行リビジョンで未実装**（フットプリントのみ）。MD応答の6番目floatは常に0前後。

## 本体⇔プローブ通信（真実源: docs/protocol-rs232c.md）
- プローブ側UART: **USART2**(IC20.9=TX=PA2 / IC20.10=RX=PA3) → MAX3232(IC21)。
- フレーム: 全コマンドは**本体→プローブ**方向。コマンド名は**大文字A-Zのみ**有効（小文字はNG返送）。行終端CR+LF。
- プローブ→本体の自律送信は**MS有効時の周期送信のみ**。

### MD応答フォーマット（§3.2.2）
- 書式: `<mV[0]>, <mV[1]>, <mV[2]>, <mV[3]>, <mV[4]>, <pressure hPa>\r\n`（6個のfloatをカンマ+スペース区切り）。
- `AD_AVE_mV[0..4]`はMCP3424の物理chと1:1ではなく、SEL3(SS)状態でindexが動的に変わる:
  - [0]=CH1, [1]=CH2, [2]=CH3(SEL3 LOW/gain x1), [3]=CH4, [4]=CH3(SEL3 HIGH/gain x2)。
- 移動平均はSADAで設定した回数で計算済み。無効ch/未更新chは0または直前値が残留。

### エラー処理（§5）
- プローブが解釈できないコマンド・不正パラメータ → `NG\r\n` 返送。
- コマンド応答タイムアウト・リトライは本体側が管理（§4）。詳細はdocs/protocol-rs232c.mdを参照。

## 補正（IM補正式作成手順.xlsx / adjust系ドキュメント — 詳細は校正時に突き合わせ）
- MLSS/SS/透視度は生mV値から補正式で物理量へ換算。ゼロ点(SADZ)・スパン(SADS)補正 + ソフト補正式。
- LED duty(SP)や移動平均(SADA)を含む調整パラメータはFlash(WPP)に保存。

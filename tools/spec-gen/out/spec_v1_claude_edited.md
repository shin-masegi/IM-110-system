# セクションA：IM-110T 本体

## A-1 はじめに

本仕様書は IM-110T 本体のプログラム動作を記述する。IM-110 は活性汚泥処理の水質を計測する **MLSS メーター** であり、操作・表示・通信・ログを担う **本体基板** と、水質計測用アナログ信号を取得する **プローブ基板** の 2 基板構成をとる。両者は RS-232C ケーブルで接続され、本体がマスターとしてプローブを制御する（通信仕様は本書セクション B および `docs/protocol-rs232c.md` を参照）。

本体基板の主要ハードウェアは以下のとおり（真実源: `CLAUDE.md` §4 / 回路図・ネットリスト）。

- MCU: **STM32L452RET6**（Cortex-M4, 80MHz, Flash 512kB / SRAM 160kB）
- 表示: **Sharp メモリ液晶 LS027B7**、**400×240 ドット・モノクロ**（SPI1、CN3 経由）
- 操作スイッチ: **電源(SW3) / 記録(SW1=PC10) / 履歴(SW2=PC11)** の 3 個。パネルシート・画面はアイコン方式
- 電源: MAX8881(U9)→3V3、LTC3221(U13) チャージポンプ→5V（プローブ給電を兼ねる）
- ストレージ / 時計: SPI2 共有バスに R2045S RTC(U8)・M95256 EEPROM(U11)
- BLE: USART1 → 74_2G125 バッファ(U7) → RN4871(U5)

本書では画面・機能ごとの動作仕様を中心に記述し、補正アルゴリズム等の技術詳細はセクション B および校正手順書に委ねる。測定項目は **MLSS / SS / 透視度 / 界面** の 4 系統である。各測定値の具体レンジ・小数桁は本体 FW と突き合わせて確定する（※要確認）。

## A-2 シナリオ

関係者の共通理解のため、典型的な利用者が IM-110 をどう使うかを 1 例示す。

活性汚泥処理場の運用担当者は、処理槽の状態を把握するために IM-110 を用いて MLSS・SS・透視度・界面を定期的に測定する。現場では、プローブを処理槽の測定対象に投入し、本体の測定画面に表示される測定値・水温をリアルタイムに確認して記録する。測定値が普段と異なる場合は、原因調査のために該当項目の校正を実施し、必要に応じてゼロ点・スパンを調整する。測定・校正の結果は本体内部（EEPROM）に記録され、BLE 経由で Reader（スマートフォンアプリ）へ読み出して運用記録・分析に活用する。

## A-3 対象外

本仕様書では以下の機能は記述・管理しない。

- 不具合の原因調査や設計途中のデータ取得に必要な機能（※一部、現時点で必須の調整機能は記載する）
- センサー生産に必要な機能（1-Wire 操作等）
- デモ器・貸出器など製品以外でその場限りに実装された機能

## A-4 概略・システム構成

本システムは **本体基板** と **プローブ基板** の 2 基板から構成される。両基板は RS-232C で接続され、本体がマスター、プローブがスレーブとして動作する。プローブはバッテリーを持たず、本体 CN4 から +5V 給電を受けて動作するため、単独では動作しない。

```
        RS-232C ケーブル（±12V level）
 ┌───────────────┐        CN4         ┌────────────────┐
 │  本体基板       │  5V/GND/RXD/TXD    │  プローブ基板     │
 │ STM32L452RET6 │──────────────────│ STM32G070KBT6  │
 │ LPUART1+MAX3232│                   │ USART2+MAX3232 │
 │ LCD/BLE/EEPROM │  ←給電 5V→         │ アナログ計測系    │
 └───────────────┘                   └────────────────┘
```

本体は操作・表示・ログ・BLE 通信・プローブ制御を担い、プローブは水質計測用アナログ信号の取得を担う。両 MCU は系列が異なる（本体=STM32L4 / プローブ=STM32G0）ため、FW・HAL・ピン配置に互換性はない。

## A-5 画面ごとの仕様

## A-6 校正

### 6.1 校正概要

校正メニューでは **MLSS / SS / 透視度** の各項目を校正でき、加えて **校正リセット**を備える。各校正は「校正待機 → 校正中 → 校正完了（異常時は校正エラー）」の流れで進む。校正はセンサーのゼロ点とスパンを調整し、測定精度を確保するための手順である。

![校正画面](assets/im-110/calib_01.png)
<sub>画面図 calib_01 — BMP版: [calib_01.bmp](assets/im-110/calib_01.bmp)</sub>

### 6.2 MLSS校正

MLSS 校正では、**ゼロ校正**（基準となる低濃度状態での基準点調整）と**スパン校正**（既知濃度での感度調整）を行い、算出した校正係数を内部メモリに記憶する。校正待機画面で記録スイッチにより校正を開始し、所定の校正入力状態で記録スイッチを押すと係数を算出する。完了画面で測定画面へ復帰する。異常時は校正エラー画面を表示する。

> ※要確認: 校正入力の与え方（標準液／ダミー入力等）、校正点、係数算出条件は本体 FW と突き合わせて確定する。AI ドラフトに含まれた DO メーター由来の記述（`O2_in`・「100%飽和」等）は IM-110 には該当しないため削除した。

### 6.3 SS校正

SS 校正の画面構成・操作は MLSS 校正（6.2）と共通で、対象項目と校正入力・係数が異なる。詳細は本体 FW と突き合わせて確定する（※要確認）。

### 6.4 透視度校正

透視度校正の画面構成・操作は 6.2 と共通で、対象項目と校正入力・係数が異なる。透視度特有の校正手順（多点校正の有無等）は本体 FW と突き合わせて確定する（※要確認）。

> ※注: `操作表示一覧` の校正シートには多数の画面図（`calib_01`〜`calib_42`）があり、基板調整（低／中／高レンジのゼロ・スパン調整、水温調整等）に相当する画面群が含まれる可能性がある。AI ドラフトはこれらを ID-200T（DO メーター）のレンジ・水温 5/20/35℃ 調整からそのまま流用したため、本節では採用せず、実画面と FW を突き合わせたうえで別途整理する。

### 6.5 校正リセット

校正リセット画面。確認表示 → 記録スイッチで実行 → 完了表示の流れで、各項目の校正係数を初期値（センサー内初期値またはプログラム内初期値）に戻す。リセット対象の範囲（校正係数のみか、他設定を含むか）は本体 FW と突き合わせて確定する（※要確認。AI ドラフトの「淡水/海水設定を初期化」は DO/塩分計向けの記述であり、IM-110 で該当するかは未確認）。

## A-8 BLE通信（Readerとの通信）

本体は RN4871 経由の BLE で Reader（スマートフォンアプリ）と通信する。

- **接続表示**: BLE 接続中は画面に BT アイコンを表示する。
- **消費電流**: BLE 通信中は消費電流が増加する（バッテリー寿命に影響）。
- **交換データ種別（概要）**: 現在の測定値（MLSS 等）・水温、校正情報、測定履歴、機器設定、機器情報（バージョン等）。

具体的な BLE コマンド・応答フォーマットは本体 FW と突き合わせて確定する（※要確認。AI ドラフトの「DO（溶存酸素）」は DO メーター由来のため IM-110 の測定値に置換した）。

## A-9 電池残量・省電力動作

- **電池残量表示**: 画面上段に電池残量アイコンを表示する。残量に応じた注意喚起・バッテリエラー表示のしきい値は本体 FW と突き合わせて確定する（※要確認）。
- **省電力動作**: 一定時間 BLE 未接続が続いた場合等に省電力状態へ移行し、消費電力を抑える。復帰条件（BLE 再接続・操作等）を含む具体仕様は本体 FW と突き合わせて確定する（※要確認。AI ドラフトの具体値（30 分・1mg/L・クロック 0.5MHz 等）は ID-200T 由来のため未採用）。

## A-10 エラー表示（本体故障・バッテリエラー）

- **本体故障**: EEPROM・RTC 等の異常を検出した場合、故障表示を行い測定動作を中止する。
- **バッテリエラー**: バッテリー異常発生時にエラー表示し、測定動作を中止する。
- **通信失敗**: BLE 通信の失敗は本体故障としては扱わない。

エラー種別ごとの表示内容・処置手順（QR コード等）の詳細は本体 FW と突き合わせて確定する（※要確認）。

# セクションB：プローブ

> プローブは**画面を持たない**。本体から RS-232C コマンドで制御されるアナログ計測モジュールである。
> 本セクションの記述は `CLAUDE.md` §4 および `docs/protocol-rs232c.md`（通信仕様の唯一の真実源）に基づく。

## B-1 概要・役割

プローブ基板は水質計測用アナログ信号を取得するモジュールであり、本体から RS-232C コマンドで制御される。電源は本体 CN4 から `VIN`（5V）供給を受け、バッテリーは搭載しないため単独では動作しない。MCU は **STM32G070KBT6**（Cortex-M0+, 64MHz, Flash 128kB / SRAM 36kB）で、本体（STM32L4）とは系列が異なる。

## B-2 ハードウェア構成（電源・MCU）

電源は本体 CN4 の `VIN`（5V）→ Q1（逆接保護, RQ5A030AP）→ `+05P` レールに入り、以下に分配される。

- MAX8881（IC14）→ `+3.3V` → フェライト → `+3.3VA`（アナログ 3.3V）
- LM27761（IC15）インバーティングチャージポンプ → `+05A` / `-05A`（アナログ ±5V）
- LM336（IC9）+ 2SC2712 → +2.5V 基準、分圧して `1V_REF` を生成

本体と異なり LTC3221 は非搭載である。

## B-3 計測フロントエンド（アナログ信号系）

- **オペアンプ**: TSX562 × 7（IC1〜IC7）。低オフセット・レール to レール CMOS。
- **アナログ MUX**: TMUX4053 × 2（IC10, IC11）。`SEL`（IC20.13/PA6）/ `SEL3`（IC20.14/PA7）でチャネル切替。
- **外部 ADC**: MCP3424（IC19）、4ch・18bit ΔΣ、I2C1 接続（SDA=IC20.1 / SCL=IC20.32）。**STM32 内蔵 ADC は計測に不使用**。
- **入力保護**: 各差動入力に 1N4148WS クランプ。
- **主要信号系ネット名**: `MLSS_JUKO` / `MLSS_REF` / `TRA_JUKO` / `TRA_REF`。

## B-4 測定チャネルとMD応答

測定データは RS-232C コマンド `MD` で 1 回取得する（`MS` 有効時は周期的に自律送信）。応答フォーマットは 6 個の float をカンマ+スペース区切りで返す。

```
<mV[0]>, <mV[1]>, <mV[2]>, <mV[3]>, <mV[4]>, <pressure hPa>\r\n
```

`AD_AVE_mV[0..4]` は MCP3424 の物理 ch と 1:1 ではなく、**`SEL3`（`SS` コマンド）の状態で index が動的に変わる**点に注意する。

| index | 対応 | 更新条件 |
|---|---|---|
| `mV[0]` | MCP3424 CH1 | `SADC` bit0=1 |
| `mV[1]` | MCP3424 CH2 | `SADC` bit1=1 |
| `mV[2]` | CH3（SEL3 LOW / gain x1） | `SADC` bit2=1 かつ `SS,0` |
| `mV[3]` | MCP3424 CH4 | `SADC` bit3=1 |
| `mV[4]` | CH3（SEL3 HIGH / gain x2） | `SADC` bit2=1 かつ `SS,1` |

- 移動平均は `SADA` で設定した回数で計算済み。
- 無効 ch / 未更新 ch は 0 または直前値が残留する。
- `<pressure hPa>`: HSPPAD143A 気圧センサ値。**現行リビジョンでは未実装のため常に 0 前後**（応答書式は変更しない）。

## B-5 RS-232Cコマンド仕様

全コマンドは **本体 → プローブ方向**のみ発行される。コマンド名は **大文字 A-Z のみ**有効（小文字は `NG\r\n` 返送）。行終端は CR+LF。プローブ → 本体の自律送信は `MS` 有効時の周期送信のみ。

| # | コマンド | パラメータ | 応答 | 機能 |
|---|---|---|---|---|
| 1 | `VR` | なし | 値応答1行 | 製品名・FWバージョン取得（例: `IM-110 Probe Ver.0.11`） |
| 2 | `SP` | X.XX（0.0〜1.0） | OK/NG | LED_REFV PWM duty 設定 |
| 3 | `SEL` | N（0,1,2） | OK/NG | SEL(PA6) 出力モード（0:OFF / 1:ON / 2:PWM） |
| 4 | `SS` | N（0,1） | OK/NG | SEL3(PA7) 出力。ADC ch3 の読取先を mV[2]（gain x1）/ mV[4]（gain x2）へ切替 |
| 5 | `MS` / `MS,N` | N（0,1）省略可 | OK | 測定データ自律送信の制御。`MS,1`=開始 / `MS,0`=停止 |
| 6 | `MD` | なし | 値応答1行 | 現在の測定値を1回送信（§B-4） |
| 7 | `SID` | N（uint32） | OK/NG | Probe ID 設定 |
| 8 | `SADZ` | N（0〜4） | OK/NG | 指定 ch の ADC ゼロ点補正 |
| 9 | `SADS` | N（0〜4） | OK/NG | 指定 ch の ADC スパン補正 |
| 10 | `SADA` | N（0〜255） | OK/NG | ADC 移動平均回数設定（既定 30） |
| 11 | `SADC` | XXXX（4文字 '0'/'1'） | OK/NG | ADC 有効 ch ビットマスク（MCP3424 CH1–4） |
| 12 | `RPP` | なし | 複数行 | Flash パラメータ一覧出力 |
| 13 | `WPP` | なし | 固定文字列 | パラメータを内蔵 Flash に保存 |
| 14 | `RPI` | なし | 固定文字列 | パラメータを初期値にリセット（RAM 上のみ） |
| M | `FUP,45063` | magic | OK→BL移行 | リペア/製造用: ROM ブートローダー移行 |

> 上記いずれにもマッチしない受信行は `NG\r\n` を返す。全コマンドの棚卸し・追加コマンド（`SSF` 等）の反映は `/sync-protocol` で別途実施する。

## B-6 ゼロ点・スパン補正（SADZ / SADS / SADA / SADC / WPP）

計測精度確保のため、以下の補正・設定を行い、確定した値を Flash に保存する。

- **`SADZ N`（ゼロ点補正）**: 指定 ch のゼロ点（計測値が 0 になるべき状態でのオフセット）を補正する。
- **`SADS N`（スパン補正）**: ゼロ点補正後、指定 ch のスパン（感度）を補正する。
- **`SADA N`（移動平均）**: ADC の移動平均回数を設定する（ノイズ低減、既定 30）。
- **`SADC XXXX`（有効 ch マスク）**: MCP3424 各 ch の有効/無効を 4bit で設定する。
- **`WPP`（Flash 保存）**: 上記調整値・LED duty(`SP`) 等を内蔵 Flash に保存し、次回起動時に反映する。`RPI` で RAM 上の値を初期化（永続化には続けて `WPP` が必要）、`RPP` で一覧出力する。

## B-7 補正式・調整パラメータ

生 mV 値から物理量（MLSS / SS / 透視度）への換算は、ベース補正（`SADZ` ゼロ点・`SADS` スパン）とソフトウェア補正式の組み合わせで行う。

- **調整パラメータ**: LED duty（`SP`, 0.0〜1.0）、移動平均回数（`SADA`, 0〜255）等。いずれも `WPP` で Flash に保存する。
- **補正の流れ**: MCP3424 から生 mV 値取得 → ゼロ点補正 → スパン補正 → ソフト補正式で物理量へ換算。

具体的な補正式・係数・調整手順は `docs/specs/IM補正式作成手順.xlsx` および校正手順書と突き合わせて確定する（※要確認）。

## B-8 エラー処理・タイムアウト

- プローブが解釈できないコマンド・不正パラメータを受信した場合、`NG\r\n` を返送する。
- コマンド応答のタイムアウト・リトライは**本体側で管理**する。
- 詳細（タイムアウト値・リトライ回数・通信切断判定）は `docs/protocol-rs232c.md` を参照。

プローブは `MD` コマンドに対し、現在の測定値を 1 回送信する。応答フォーマットは次のとおりである。

```
<mV[0]>, <mV[1]>, <mV[2]>, <mV[3]>, <mV[4]>, <pressure hPa>\r\n
```

6 個の `float` をカンマとスペースで区切って送信する。前半 5 個は内部配列 `AD_AVE_mV[5]`（単位 mV）、6 番目は気圧センサ値である。移動平均は `SADA` コマンドで設定した回数で計算済みの値が入る。この書式は測定データ自動送信（MS 有効時の周期送信）でも同一である。

`AD_AVE_mV[]` の各 index は MCP3424 の物理チャネルと 1 対 1 に対応するのではなく、`SEL3`（`SS` コマンド）の状態によって index が動的に変わる点に注意する。各要素の対応は以下のとおりである。

- `AD_AVE_mV[0]`: MCP3424 CH1（`SADC` の bit0 が 1 のとき更新）
- `AD_AVE_mV[1]`: MCP3424 CH2（`SADC` の bit1 が 1 のとき更新）
- `AD_AVE_mV[2]`: MCP3424 CH3 の SEL3 = LOW 時（`SADC` の bit2 が 1 かつ `SS,0`）。通常ゲイン x1 の測定
- `AD_AVE_mV[3]`: MCP3424 CH4（`SADC` の bit3 が 1 のとき更新）
- `AD_AVE_mV[4]`: MCP3424 CH3 の SEL3 = HIGH 時（`SADC` の bit2 が 1 かつ `SS,1`）。同一の CH3 を PGA gain x2 で再測定した値

すなわち `SS,0` と `SS,1` を切り替えると、同じ物理 CH3 の読み取り値が `[2]` と `[4]` のどちらへ流れるかが切り替わる。更新されなかった側には古い値が残留する。一度も有効化されていないチャネルはゼロ初期化により `0.000000` であり、一度有効化された後に無効化されたチャネルは直前の値が残留する。

6 番目の気圧（pressure hPa）は、HSPPAD143A 気圧センサー（I2C2 / PA11=SCL, PA12=SDA）の実測値である。現行リビジョンでは実装済み・稼働中であり、周期取得した hPa を返す。`MD` 応答書式は固定であり、6 番目の float に実測 hPa が入る。なお HSPPAD143A 内蔵の温度センサ値は `MD` 応答には採用しない方針である。
